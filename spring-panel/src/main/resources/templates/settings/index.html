      Профили прокси и VPN

      Создайте именованные профили маршрутизации, чтобы потом выбирать их для проекта, подсистемы ботов и отдельных интеграций.

      Профиль Тип Маршрут Статус Действия
      Профили нужны, чтобы не копировать одинаковые прокси/VPN вручную для каждого маршрута.
      Для VLESS создайте несколько профилей (host, port, token), затем выберите их цепочкой в маршруте проекта/ботов. Первый профиль в списке — основной, остальные используются как резерв.

      Сетевые маршруты интеграций

      Задайте отдельные прокси/VPN для всего проекта и отдельно для подсистемы ботов. Можно использовать как ручные настройки, так и готовые профили.

      Весь проект
      Базовый маршрут для панели и интеграций. Если ниже для ботов выбран режим «Наследовать проект», будет использован именно этот блок.
      Профили можно выбирать обычными кликами. Повторный клик снимает выбор.
      Первый профиль — основной, дальше резервные. При проблеме соединения система переключится на следующий доступный профиль.
      Цепочка пока не выбрана.
      Если маршрут не прошёл проверку доступности, он исключается на это время перед новой попыткой.
      Сначала выберите режим. Для отдельного прокси заполните host/port, для VPN — название, endpoint и путь к конфигу.
      Подсистема ботов
      Отдельный маршрут только для Java/VK-процессов ботов. Удобно, если панель должна работать напрямую, а боты — через выделенный прокси или VPN.
      Профили можно выбирать обычными кликами. Повторный клик снимает выбор.
      Можно выбрать отдельную цепочку профилей только для процессов ботов.
      Цепочка пока не выбрана.
      Недоступный маршрут временно исключается и бот-процессы переключаются на следующий из списка.
      Если выбрать «Наследовать проект», боты будут использовать маршрут из левого блока без дублирования настроек.
      Прокси применяются к запросам панели и автоматически передаются отдельным процессам ботов. VPN сохраняется как конфигурация маршрута для окружения и конкретных ботов.

      Добавить канал
      Что понадобится
      • Для Telegram — токен из @BotFather и название канала для панели.
      • Для VK — токен сообщества (доступ к сообщениям), ID сообщества, код подтверждения Callback API и при необходимости секретный ключ.
      • Для MAX — bot token, название канала в панели и при необходимости webhook secret.
      Проверяется автоматически для Telegram.
      Отображается в списке обращений клиентов.
      Опционально. Можно оставить пустым — бот сохранит ID после добавления в чат.
      Нужно только если используете Bot API mirror или self-hosted endpoint. Это не proxy: панель и бот будут обращаться к указанному `/bot...` URL напрямую.
      Найдите в настройках сообщества → Работа с API.
      Настройка VK Callback API
      1. Создайте ключ доступа сообщества с правами сообщений: «Работа с API» → «Ключи доступа».
      2. В «Callback API» укажите адрес вида https://ваш-домен/webhooks/vk/<group_id>.
      3. Вставьте код подтверждения и при необходимости секретный ключ, чтобы включить проверку запросов.
      4. Активируйте события сообщений и перед запуском сервиса выполните python vk_webhook_support.py.
      Поле используется только как запасной параметр для webhook-режима (fallback).
      Как подключить MAX
      1. Получите bot token в кабинете интеграций MAX и вставьте его в поле «Токен MAX-бота».
      2. Бот работает через long polling: отдельный webhook endpoint для MAX настраивать не нужно.
      3. Если позже потребуется вернуться к webhook-режиму, secret можно сохранить заранее в этом блоке.
      Для Telegram токен проверяется через getMe. Для VK и MAX токен обязателен для рабочего запуска бота.
