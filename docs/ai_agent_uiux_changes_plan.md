# План Изменений: AI Agent + UI/UX Упрощение

## Цель
- Уменьшить визуальную перегруженность страницы диалогов.
- Вынести операционный контроль AI в отдельную страницу.
- Сохранить прозрачность развития агента через фиксированный чек-лист.

## Что внедряем в этой итерации
- [x] Отдельная страница `AI Ops` (`/ai-ops`) с KPI, алертами, runbook и журналом событий.
- [x] Навигационный пункт `AI Ops` в сайдбаре.
- [x] `Compact mode` для страницы диалогов с сохранением выбора в `localStorage`.
- [x] Фильтрация событий AI по `eventType` и `actor`.
- [x] Экспорт событий AI в CSV.

## Архитектурные изменения
1. Routing:
   - `DialogsController` -> `GET /ai-ops` -> шаблон `dialogs/ai-ops`.
2. Навигация:
   - `fragments/navbar.html` -> добавлен пункт `AI Ops`.
3. UI Диалогов:
   - `dialogs/index.html` -> добавлен переключатель `Compact mode`.
   - `dialogs.js` -> логика `loadCompactMode/toggleCompactMode`.
4. UI AI Ops:
   - новый шаблон `dialogs/ai-ops.html`.
   - новый скрипт `static/js/ai-ops.js`.

## Критерии успеха
- Сокращение среднего времени до первого ответа оператора.
- Сокращение количества кликов до отправки ответа.
- Рост доли использования AI-подсказок без роста correction rate.
