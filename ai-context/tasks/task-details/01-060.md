# Задача [01-060]

Переключить целевые лицензии RMS по типу сервера и показать kiosk-счётчик.

## Цель

Сделать лицензионный мониторинг RMS тип-зависимым: для `IIKO_CHAIN` считать основной лицензией `FullEdition (Server)` с `id=10`, а для `IIKO_RMS` дополнительно показывать количество лицензий `iikoConnector for Get Kiosk (iikoFront)` с `id=36073118`.

## Контекст

- До изменения RMS-мониторинг ориентировался на одну целевую лицензию `RMS (Front Fast Food)` с `id=100`.
- Теперь правило зависит от `server_type`.
- Общий список RMS уже умеет показывать количество целевой лицензии, осталось сделать label/id динамическими и добавить kiosk-count для `IIKO_RMS`.

## Что сделать

- В `RmsLicenseMonitoringService` добавить тип-зависимый выбор основной лицензии.
- Для `IIKO_RMS` добавить отдельный расчёт количества лицензий `iikoConnector for Get Kiosk (iikoFront)`.
- Вернуть в DTO RMS динамический label/id основной лицензии и kiosk-счётчик.
- Обновить общий список RMS, чтобы он показывал правильную лицензию по типу сервера и, при наличии, kiosk-count.

## Критерии готовности

- Для `IIKO_CHAIN` срок и количество считаются по `FullEdition (Server)` с `id=10`.
- Для остальных RMS базовая логика по `RMS (Front Fast Food)` с `id=100` сохранена.
- Для `IIKO_RMS` в общем списке показывается дополнительное количество лицензий `iikoConnector for Get Kiosk` с `id=36073118`, если они присутствуют.
