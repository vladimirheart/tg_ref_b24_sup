# Задача [01-046]

## Короткое описание

Вынести все данные SSL- и RMS-мониторинга из primary SQLite БД панели в отдельную `monitoring.db`, сохранив работоспособность страниц мониторинга и мигрировав уже накопленные данные.

## Что нужно сделать

- Добавить отдельный datasource для `monitoring.db`.
- Перевести SSL- и RMS-хранилища на отдельный monitoring storage.
- Не ломать остальной JPA-контур панели и не переносить в новую БД несвязанные сущности.
- Создать bootstrap/migration слой для таблиц мониторинга.
- Перенести уже существующие записи SSL и RMS из старой primary DB в `monitoring.db`.
- Сделать миграцию идемпотентной, без повторного создания дублей.

## Связанные файлы

- `spring-panel/src/main/resources/application.yml`
- `spring-panel/src/main/java/com/example/panel/config/`
- `spring-panel/src/main/java/com/example/panel/repository/`
- `spring-panel/src/main/java/com/example/panel/service/SslCertificateMonitoringService.java`
- `spring-panel/src/main/java/com/example/panel/service/RmsLicenseMonitoringService.java`
- `spring-panel/src/main/java/com/example/panel/service/RmsMonitoringSeedImportService.java`

## Ограничения

- `monitoring.db` должна хранить именно данные мониторинга, а не общий runtime панели.
- Страницы мониторинга должны продолжить работать без изменения API.
- Уже импортированные RMS-записи и существующие SSL-записи нельзя потерять.
