# Задача [01-028]

## Заголовок

Разделить AI orchestration на специализированные сервисы и исправить learning semantics

## Контекст

`DialogAiAssistantService` перегружен разнородной логикой: нормализация,
retrieval, policy, decision, learning и monitoring. Это усложняет поддержку и
вносит неявные побочные эффекты в самообучение.

## Что нужно сделать

- разделить текущий монолитный сервис на слои:
  `AiInputNormalizerService`, `AiRetrievalService`, `AiPolicyService`,
  `AiDecisionService`, `AiLearningService`, `AiMonitoringService`;
- оставить `DialogAiAssistantService` как orchestration layer;
- исправить learning flow:
  без подтвержденного intent/scope запись идет только в `draft`;
- запретить auto-promote из операторского ответа без explicit review approve;
- обновить integration tests на новый orchestration flow.

## Критерии готовности

- orchestration и бизнес-слои разделены с явными контрактами;
- обучение не повышает trust автоматически без review;
- решение по диалогу проходит единый конвейер normalize → policy → retrieval →
  decision;
- регрессии по существующим AI runtime сценариям отсутствуют.
