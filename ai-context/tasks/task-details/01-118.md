# Задача [01-118]

## Короткое описание

Уплотнить dashboard и исправить группировки блоков по каналам, бизнесам и сетям.

## Цель

Сделать верхнюю часть дашборда компактнее и привести смысл трёх аналитических блоков к ожидаемой бизнес-логике: каналы, бизнесы и типы сетей.

## Контекст

- Пользователь указал, что блоки `Операционный центр поддержки iguana` и `Ключевые метрики смены` визуально слишком крупные.
- Фильтр-секция занимает слишком много места и должна быть более компактной.
- В `spring-panel/src/main/resources/templates/dashboard/index.html` блок `Заявки по каналам` сейчас получает данные не из основного dashboard payload, а отдельным запросом к `/stats_data`.
- В `spring-panel/src/main/java/com/example/panel/service/DashboardAnalyticsService.java` текущая агрегация `business` и `network` не соответствует ожиданиям:
  `business` должно агрегироваться только по бизнесу, а `network` должно показывать `Корпоративная сеть` и `Партнёры-франчайзи`.
- Источник истины для разбивки по бизнесу и типу сети находится в `config/shared/locations.json`.

## Что сделать

- Исправить server-side агрегацию dashboard так, чтобы `По бизнесу` считался по бизнесам, а `По сетям` по типам партнёрской структуры из `locations.json`.
- Перевести блок `Заявки по каналам` на отображение данных, которые сейчас выводятся в `По сетям`, без отдельного запроса к `/stats_data`.
- Уплотнить hero-блок, секцию ключевых метрик и фильтры в шаблоне dashboard.

## Критерии готовности

- Верхняя часть dashboard занимает заметно меньше вертикального пространства.
- Блок `По бизнесу` больше не дублирует `БлинБери` и `СушиВёсла` из-за разделения по каналам.
- Блок `По сетям` показывает разбивку на `Корпоративная сеть` и `Партнёры-франчайзи`.
- Блок `Заявки по каналам` показывает набор данных, который ранее был в `По сетям`.

## Обязательные дополнительные условия

- В финальном ответе подготовить `Резюме для коммита` по формату из `ai-context/baseline/guides/tasks.md`.
- После file changes создать отдельный changelog-файл в `ai-context/changelog` и включить в него пользовательский промт, который привёл к изменению.
