# Задача [01-027]

## Заголовок

Ввести trust/freshness модель памяти и запретить auto-reply из слабых источников

## Контекст

Сейчас источники retrieval используются почти одинаково, что повышает риск
небезопасных auto-reply. Память также недостаточно описывает актуальность,
владельца и область применимости решений.

## Что нужно сделать

- ввести правила доверия к источникам:
  `memory/knowledge` допустимы для auto-reply, `tasks` только assist-only,
  `history/applicant_history` только suggestion;
- расширить `ai_agent_solution_memory` полями:
  `status`, `trust_level`, `intent_key`, `slot_signature`, `scope_*`,
  `safety_level`, `source_type`, `last_verified_at`, `expires_at`, `verified_by`;
- формализовать статусы памяти:
  `draft`, `approved`, `deprecated`, `rejected`;
- расширить `ai_agent_event_log` policy/trace полями для explainability;
- добавить таблицу `ai_agent_intent_policy` для явной маршрутизации по intent.

## Критерии готовности

- источник кандидата влияет на право auto-reply по явным правилам;
- stale/непроверенные записи памяти не используются как trusted evidence;
- для памяти есть жизненный цикл и возможность безопасного deprecate/reject;
- решения агента содержат достаточный source trace для разбора.
