# Задача [01-014]

## Короткое описание

Довести MAX-диалоги до корректного runtime: убрать лишние ACK, связать автозакрытие с настройками и включить media reply.

## Цель

- Не отправлять клиенту в MAX лишнее служебное подтверждение после каждого сообщения в активную заявку.
- Привязать авто-закрытие в `java-bot` к реальному `auto_close_hours` из shared settings.
- Считать исходящие ответы оператора активностью диалога, чтобы заявка не закрывалась из-за ответа из панели.
- Разрешить отправку медиа в MAX из composer и через paste из буфера обмена.

## Что сделано

- В `MaxWebhookController` убран клиентский ACK `"Сообщение добавлено в заявку..."` для активной заявки.
- В `MaintenanceTasks`:
  - добавлено чтение `auto_close_hours` из `config/shared/settings.json`;
  - `0` теперь трактуется как отключение автозакрытия;
  - при отсутствии корректной настройки сохраняется fallback `24h`.
- В `DialogReplyService`:
  - после успешного исходящего ответа обновляется `ticket_active.last_seen`;
  - добавлена отправка медиа в MAX через цепочку `POST /uploads -> upload binary -> POST /messages`;
  - ограничения на media reply больше не режут MAX-каналы.
- Добавлен `MaintenanceTasksTest` на чтение настройки и поведение при `auto_close_hours=0`.
