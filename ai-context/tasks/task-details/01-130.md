# Задача [01-130]

## Короткое описание

Разделить giant dialogs.js на bounded client-side runtime modules.

## Цель

Убрать из `dialogs.js` роль client-side monolith и перевести runtime диалогов
на набор модулей с явными границами между list/details/workspace/actions/AI и
notification loops.

## Контекст

- По audit-срезу `dialogs.js` держит около `10426` строк.
- Внутри файла смешаны list/filter polling, details/history, workspace shell,
  quick actions, macro workflow, AI ops/review и bell refresh semantics.
- Backend домена `dialogs` уже существенно разрезан; следующий practical risk
  живёт в browser runtime, а не в `DialogService`.
- При этом вокруг `dialogs.js` уже есть накопленный runtime/UI backlog:
  `01-090`, `01-091`, `01-101`, `01-102`, `01-103`, `01-104`, `01-109`,
  `01-110`, `01-117`; эти задачи задают compatibility baseline для legacy
  details modal, history/media preview, my dialogs, participants/reassign,
  reply transport и pagination.

## Пересечения с backlog

- `01-090` / `01-101` / `01-102` — baseline для legacy details modal,
  category modal, media preview и scroll containment.
- `01-091` / `01-117` — baseline для list/my-dialogs/pagination behavior.
- `01-103` / `01-104` — participant and reassignment UX/runtime нельзя ломать
  при split `workspace` и `quick actions` модулей.
- `01-109` / `01-110` — reply transport и spam action flow считаются текущим
  runtime baseline для details/workspace actions.

## Что сделать

- Выделить shared client boundary для API helpers, polling timers, route/open
  state и refresh bus.
- Разрезать `dialogs.js` минимум на модули `list+filters`, `details+history`,
  `workspace shell`, `quick actions`, `macro workflow`, `AI ops/review`,
  `notifications refresh`.
- Убрать зависимости между соседними consumer surfaces через случайные global
  helpers там, где это возможно без переписывания API.
- Зафиксировать migration order и compatibility net так, чтобы сначала
  выезжали менее рискованные shared/list modules, а history/details/workspace
  flows переезжали без потери текущих modal/action contracts.
- Сохранить текущие runtime contracts между `/api/dialogs`, `details`,
  `history`, `workspace` и bell loop.

## Текущий прогресс

- На первом execution-срезе вынесены bounded runtime-модули
  `dialogs-list-runtime.js` и `dialogs-ai-runtime.js`.
- `dialogs/index.html` теперь подключает их отдельно, а `dialogs.js` переведён
  на thin wrapper/orchestration для list/filter/pagination/bulk и
  AI ops/review/monitoring сценариев.
- Сохранён compatibility corridor для `my_dialogs`, pagination, bulk actions,
  AI suggestions, AI review queue и AI monitoring summary.
- На текущем проходе добавлен bounded runtime
  `dialogs-details-history-runtime.js`, куда вынесены history polling,
  previous-history batches, media preview/audio handling, media send helpers и
  history rendering/filtering для details и workspace surfaces.
- `dialogs.js` дополнительно переведён на thin wrappers для
  `details+history/media`, а legacy-файл теперь в большей степени удерживает
  orchestration и modal/action wiring вместо прямого владения history state.
- Следующим bounded support-slice вынесен
  `dialogs-workspace-runtime.js`, который забрал workspace reply target,
  draft lifecycle, message pagination/load-more, partial section reload,
  inline navigation и active-contract refresh helpers.
- На следующем execution-срезе в `dialogs-workspace-runtime.js` также
  перенесён `renderWorkspaceShell`, включая messages/history/events/SLA shell
  rendering и связку composer readonly/navigation orchestration.
- Следующим шагом в тот же `dialogs-workspace-runtime.js` перенесён
  `workspace client/context render`:
  `renderWorkspaceClientProfile`, contract/profile helper'ы и extra-attribute
  formatting больше не живут как primary render-код в `dialogs.js`.
- Следующим mini-slice в тот же runtime перенесены `workspace banners/readonly`:
  rollout/parity banners и readonly-policy helpers больше не являются
  primary UI-state кодом внутри `dialogs.js`.
- `dialogs.js` теперь использует thin delegates и для workspace
  banners/readonly helpers, а в legacy-файле остаются в основном
  categories/actions wiring, macro/telemetry и соседняя orchestration-логика.
- Оставшиеся поддомены для следующих проходов: `quick actions`,
  `macro workflow`, `notifications refresh` и дочистка
  remaining orchestration drift вокруг legacy details/workspace modal flows.

## Критерии готовности

- `dialogs.js` больше не является единственным giant runtime entrypoint.
- Основные dialog UI-сценарии разложены по bounded client-side модулям.
- Поведение quick actions, workspace, history и notifications сохраняется без
  регрессии.
