# Задача [01-130]

## Короткое описание

Разделить giant dialogs.js на bounded client-side runtime modules.

## Цель

Убрать из `dialogs.js` роль client-side monolith и перевести runtime диалогов
на набор модулей с явными границами между list/details/workspace/actions/AI и
notification loops.

## Контекст

- По audit-срезу `dialogs.js` держит около `10426` строк.
- Внутри файла смешаны list/filter polling, details/history, workspace shell,
  quick actions, macro workflow, AI ops/review и bell refresh semantics.
- Backend домена `dialogs` уже существенно разрезан; следующий practical risk
  живёт в browser runtime, а не в `DialogService`.
- При этом вокруг `dialogs.js` уже есть накопленный runtime/UI backlog:
  `01-090`, `01-091`, `01-101`, `01-102`, `01-103`, `01-104`, `01-109`,
  `01-110`, `01-117`; эти задачи задают compatibility baseline для legacy
  details modal, history/media preview, my dialogs, participants/reassign,
  reply transport и pagination.

## Пересечения с backlog

- `01-090` / `01-101` / `01-102` — baseline для legacy details modal,
  category modal, media preview и scroll containment.
- `01-091` / `01-117` — baseline для list/my-dialogs/pagination behavior.
- `01-103` / `01-104` — participant and reassignment UX/runtime нельзя ломать
  при split `workspace` и `quick actions` модулей.
- `01-109` / `01-110` — reply transport и spam action flow считаются текущим
  runtime baseline для details/workspace actions.

## Что сделать

- Выделить shared client boundary для API helpers, polling timers, route/open
  state и refresh bus.
- Разрезать `dialogs.js` минимум на модули `list+filters`, `details+history`,
  `workspace shell`, `quick actions`, `macro workflow`, `AI ops/review`,
  `notifications refresh`.
- Убрать зависимости между соседними consumer surfaces через случайные global
  helpers там, где это возможно без переписывания API.
- Зафиксировать migration order и compatibility net так, чтобы сначала
  выезжали менее рискованные shared/list modules, а history/details/workspace
  flows переезжали без потери текущих modal/action contracts.
- Сохранить текущие runtime contracts между `/api/dialogs`, `details`,
  `history`, `workspace` и bell loop.

## Текущий прогресс

- Исходный audit baseline был около `10426` строк в `dialogs.js`; после
  текущих выносов основной runtime entrypoint сжат примерно до `4179` строк.
- В отдельные bounded runtime-модули уже вынесены
  `dialogs-list-runtime.js`, `dialogs-ai-runtime.js` и
  `dialogs-details-history-runtime.js`, поэтому list/filter/pagination/bulk,
  AI ops/review/monitoring и history/media больше не живут как primary inline
  слои giant-файла.
- `dialogs-workspace-runtime.js` теперь держит workspace shell и support-flow:
  draft lifecycle, reply target, message pagination/load-more, partial reload,
  inline navigation, active-contract refresh, client/context render и
  readonly/rollout/parity banners.
- Соседние UI-кластеры тоже уже разнесены по bounded runtime'ам:
  `dialogs-actions-runtime.js`, `dialogs-macro-runtime.js`,
  `dialogs-notifications-runtime.js`, `dialogs-templates-runtime.js` и
  `dialogs-flow-runtime.js`.
- На текущем проходе добавлен `dialogs-experiment-runtime.js`, который забрал
  experiment info panel, telemetry summary/guardrails, rollout packet,
  scorecard/decision и auto-refresh wiring для experiment modal.
- Следующим bounded support-slice добавлен `dialogs-shell-runtime.js`, который
  забрал modal safety helpers, fallback dismiss/scroll containment, avatar
  hydration helpers и единый `openDialogSurface` для page-shell orchestration.
- Следующим bounded support-slice добавлен `dialogs-participants-runtime.js`,
  который забрал participants/reassign state, rendering, operator loading и
  event wiring для details/workspace participant-management flows.
- Следующим bounded details-slice добавлен `dialogs-details-runtime.js`,
  который забрал details open-flow, status/responsible summary rendering и
  time-metrics orchestration для legacy details modal.
- Следующим support-slice добавлен `dialogs-my-dialogs-runtime.js`, который
  забрал `my dialogs` state normalization, panel rendering и click-wiring для
  owner-focused list surface.
- Следующим shared helper-slice добавлен `dialogs-avatar-runtime.js`, который
  забрал responsible/message avatar rendering и avatar-spec resolution для
  details/history/workspace surface'ов.
- Следующим list helper-slice добавлен `dialogs-sla-runtime.js`, который
  забрал SLA badge calculation/rendering для dialog rows и связанный
  update-loop вокруг list surface.
- Следующим presentation helper-slice добавлен `dialogs-presentation-runtime.js`,
  который забрал channel/rating/meta formatting и `renderWorkspaceMessageItem`
  для shared workspace/list presentation surface.
- Следующим более широким workspace-slice сам `dialogs-workspace-runtime.js`
  забрал partial reload helpers, categories rendering/state messaging,
  workspace SLA label formatting и весь пакет context/parity gap telemetry.
- Следующим более широким workspace open-slice сам
  `dialogs-workspace-runtime.js` забрал route/cooldown fallback orchestration:
  initial-route reload, failure streak, cooldown и workspace-to-legacy
  fallback/open flow.
- Следующим более широким workspace telemetry/contract-slice сам
  `dialogs-workspace-runtime.js` забрал workspace telemetry transport,
  disclosure-wiring, contract preload/validation и fallback-reason resolver,
  поэтому `dialogs.js` больше не держит этот transport/guardrail cluster
  inline.
- Следующим более широким templates/category-slice сам
  `dialogs-templates-runtime.js` забрал category badge rendering,
  details/workspace category summary sync и debounced categories save, поэтому
  `dialogs.js` больше не держит этот details/workspace categories cluster
  inline.
- Следующим более широким actions continuity-slice сам
  `dialogs-actions-runtime.js` забрал row status/responsible continuity,
  details reply send/paste wiring и workspace manual legacy-open policy flow,
  поэтому `dialogs.js` больше не держит этот action-layer cluster inline.
- Следующим более широким details-history interaction-slice сам
  `dialogs-details-history-runtime.js` забрал details history menu wiring,
  reply-target state, edit/delete/reply actions и details media/preview
  interaction binding, поэтому `dialogs.js` больше не держит этот
  history interaction cluster inline.
- Следующим небольшим notifications delegate cleanup шагом `dialogs.js`
  перестал держать даже тонкие notification wrappers: соседние runtime'ы
  теперь подключаются к `dialogs-notifications-runtime.js` напрямую.
- Следующим более крупным workspace interaction-slice сам
  `dialogs-workspace-runtime.js` забрал retry/load-more wiring, workspace
  message reply/media interaction и composer draft/send shortcut bindings,
  поэтому `dialogs.js` больше не держит этот workspace event-layer inline.
- Следующим bounded shell/layout-slice сам `dialogs-shell-runtime.js` забрал
  compact/list-only mode persistence, column width persistence/restore и
  column/details resize wiring, поэтому `dialogs.js` больше не держит этот
  page-shell layout cluster inline.
- Следующим небольшим presentation-formatting шагом сам
  `dialogs-presentation-runtime.js` забрал `date/sender` formatting helpers:
  `normalizeMessageSenderByType`, `resolveSenderLabel`,
  `parseUtcDateValue`, `formatUtcDate` и `formatTimestamp`, поэтому
  `dialogs.js` больше не держит этот shared formatting cluster inline.
- Следующим небольшим shell task-navigation шагом сам
  `dialogs-shell-runtime.js` забрал `setTaskDraft`, `buildTaskCreateUrl` и
  единый `openTaskCreateSurface`, поэтому `dialogs.js` больше не держит этот
  task draft/navigation cluster inline.
- Следующим более крупным list preferences-slice сам
  `dialogs-list-runtime.js` забрал server-backed triage preferences:
  `loadPageSize`, `configureSlaWindowSelect`, `persistPageSize`,
  `restoreDialogPreferences`, `persistDialogPreferences`,
  `loadServerTriagePreferences` и внутренний debounce/save state, поэтому
  `dialogs.js` больше не держит этот list preference/state cluster inline.
- `dialogs/index.html` теперь подключает runtime-модули отдельно, а
  `dialogs.js` всё больше работает как thin orchestration/delegate слой,
  удерживающий compatibility wiring между уже вынесенными bounded surface'ами.
- Remaining focus для следующих проходов: зачистка остаточных
  тонких list/shared helper'ов (`normalizePageSize`, `normalizeDialogView`,
  column/view-state glue, если вынос ещё оправдан), а затем финальный
  regression pass вокруг уже вынесенных
  details/shell/avatar/my-dialogs/sla/presentation/workspace/templates/actions/
  details-history runtime-модулей, без возврата transport/route/cooldown/
  category-save/actions/history-interaction/notification-wrapper/workspace-event
  логики обратно в `dialogs.js`.

## Критерии готовности

- `dialogs.js` больше не является единственным giant runtime entrypoint.
- Основные dialog UI-сценарии разложены по bounded client-side модулям.
- Поведение quick actions, workspace, history и notifications сохраняется без
  регрессии.
