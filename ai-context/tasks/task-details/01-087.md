# Задача [01-087]

Починить открытие модалки пользователя в настройках панели: при редактировании существующей учётки и при создании новой страница уходит в неактивное состояние, как будто форму перекрывает другой слой.

## Цель

- вернуть нормальное открытие модалки пользователя поверх окна `usersModal`;
- устранить блокировку интерфейса при редактировании и создании пользователя;
- применить фикс в общем `auth-management`, чтобы он покрывал все дочерние пользовательские модалки.

## Контекст

- управление пользователями работает внутри `usersModal` на странице настроек;
- форма пользователя, смена пароля, предпросмотр фото и состав ветки открываются как дочерние модалки;
- похожая проблема уже была у других вложенных форм и решалась переносом модалки в `document.body`.

## Что сделать

- вынести дочерние модалки `auth-management` в `document.body` до инициализации Bootstrap instances;
- не менять API и бизнес-логику управления пользователями;
- проверить diff и убедиться, что фикс локализован в фронтенд-слое.

## Критерии готовности

- открытие редактирования пользователя не делает страницу “мёртвой”;
- создание нового пользователя открывает видимую рабочую модалку;
- дочерние модалки `auth-management` не конфликтуют по слоям с `usersModal`.

## Дополнительные условия

- после file changes создать отдельный changelog в `ai-context/changelog`;
- в финальном ответе подготовить `Резюме для коммита`.
