# Задача [01-073]

## Короткое описание

Починить сохранение локаций при дублях `city/value` в `settings_parameters`.

## Цель

Сделать persistence location-структуры устойчивым к ситуациям, когда разные ветки дерева
используют одинаковые названия города или значения `value`, чтобы сохранение и синхронизация не
перетирали соседние записи.

## Контекст

- Location-дерево уже перестало быть плоским набором параметров: в effective payload участвуют
  бизнес, тип партнёра, страна, город, локации и live-департаменты.
- При такой модели одинаковые `city`/`value` могут легально встречаться в разных ветках, и
  простой поиск записи по одному текстовому значению становится рискованным.
- В текущем backlog это один из ключевых data-integrity baseline-пакетов для `Track A3`,
  потому что без него дальнейший runtime split только разнесёт старую ошибку по модулям.
- Смежные consumers вроде `01-092` зависят от того, что effective location catalog не портится
  при сохранении из настроек.

## Пересечения с backlog

- `01-071` / `01-072` задают source boundary и editor boundary; `01-073` закрывает data-persistence
  риск внутри того же location-контура.
- `01-074` / `01-077` зависят от того, что ручной sync может безопасно мёржить результат даже при
  дублях в человекочитаемых значениях.
- `01-128` / `01-129` не должны повторно вносить save-логику, которая опирается только на `value`
  без дополнительного identity-контекста.

## Что сделать

- Пересмотреть save/update flow для location-структуры в `settings_parameters`.
- Убрать привязку к недостаточно уникальным `city` / `value` как к единственному идентификатору записи.
- Сохранить возможность держать одинаковые человекочитаемые значения в разных ветках location-дерева.
- Зафиксировать regression net на сохранение и повторное чтение effective locations payload.

## Критерии готовности

- Сохранение location-структуры не перетирает соседние записи при дублях `city` или `value`.
- Effective payload после сохранения сохраняет корректные ветки и связи.
- Повторное открытие блока `Структура локаций` показывает ту же структуру, которую пользователь сохранил.
