# Задача [01-121]

## Короткое описание

Вынести рабочее время dashboard в настройки каналов и настраивать его отдельно для каждого бота.

## Цель

Сделать интервал рабочего времени для heatmap и почасовой нагрузки на dashboard настраиваемым по каждому каналу прямо на странице настроек, чтобы визуализация больше не зависела от общего хардкода `09:00 - 18:00`.

## Контекст

- Пользователь подтвердил, что настройка рабочего времени нужна, и уточнил, что её надо задавать отдельно для каждого канала на странице настроек в блоке `Каналы (боты)`.
- Текущая реализация heatmap на dashboard уже использует `activity_stats`, но рабочие часы там пока захардкожены.
- API каналов уже умеет сохранять произвольные `delivery_settings`, поэтому отдельная таблица или миграция не требуются.

## Что сделать

- Добавить в редактор канала на странице настроек поля начала и конца рабочего времени и сохранять их в `delivery_settings`.
- На сервере читать рабочие часы по `channelId` из `delivery_settings` и подставлять их в `activity_stats` для dashboard.
- На dashboard убрать хардкод легенды рабочего времени и показывать реальные значения из payload, включая пояснение при смешанной выборке каналов.

## Критерии готовности

- Для каждого канала в блоке `Каналы (боты)` можно отдельно задать начало и конец рабочего времени.
- Heatmap и подпись рабочего времени на dashboard используют сохранённые часы канала, а не фиксированные `09:00 - 18:00`.
- При смешанной выборке на dashboard есть понятный источник интервала рабочего времени, если он взят по основному каналу среза.
