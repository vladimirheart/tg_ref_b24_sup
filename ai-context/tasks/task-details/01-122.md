# Задача [01-122]

## Короткое описание

Починить сборку spring-panel после добавления рабочих часов каналов на dashboard.

## Цель

Убрать регрессию компиляции, чтобы `spring-panel` снова запускался через `run-windows.bat` и собирался после изменений в `DashboardAnalyticsService`.

## Контекст

- Пользователь приложил лог запуска `run-windows.bat`, где сборка падает на `DashboardAnalyticsService.java`.
- Ошибка компиляции связана с `Map.of(...)` в `ActivityStats.toMap()`: после добавления новых полей туда стало передаваться больше 10 пар ключ-значение.
- Поведение dashboard менять не требуется; нужен только безопасный фикс структуры `toMap()`.

## Что сделать

- Заменить проблемный `Map.of(...)` в `ActivityStats.toMap()` на реализацию без ограничения по числу полей.
- Пересобрать `spring-panel` и убедиться, что компиляция снова проходит.
- Зафиксировать исправление отдельным changelog-файлом.

## Критерии готовности

- `spring-panel` успешно компилируется после исправления.
- `ActivityStats.toMap()` больше не использует форму `Map.of(...)`, которая ограничивает число пар.

## Обязательные дополнительные условия

- В финальном ответе подготовить `Резюме для коммита` по формату из `ai-context/baseline/guides/tasks.md`.
- После file changes создать отдельный changelog-файл в `ai-context/changelog` и включить в него пользовательский промт, который привёл к изменению.
