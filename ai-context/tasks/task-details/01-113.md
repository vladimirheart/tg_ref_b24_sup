# Задача [01-113]

## Короткое описание

Привести основные модалки страницы настроек к единому стилю открытия по
образцу «Каналы (боты)».

## Цель

Сделать primary-модалки settings единообразными: одинаковая подача, lead-блок,
ритм открытия и рабочее ощущение “control panel”, а не набора случайных окон.

## Контекст

- На странице настроек есть несколько крупных primary-модалок:
  `channelsModal`, `categoriesModal`, `locationsModal`, `parametersModal`,
  `reportingModal`, `managerBindingsModal`, `usersModal` и другие.
- Канальный блок был принят как более удачный образец UX для основных settings
  surfaces.
- Эта задача теперь служит UI baseline для последующего page-shell/runtime
  split в `01-128` и `01-129`.

## Что сделать

- Выровнять primary settings-modals по общей схеме открытия и composition.
- Использовать единый visual/structural pattern для header, lead block,
  body-shell и footer действий.
- Сохранить предметные различия модалок, не превращая их в безликие копии.

## Критерии готовности

- Основные settings-modals открываются и воспринимаются как единая система.
- Паттерн `Каналы (боты)` используется как baseline для остальных крупных
  модалок.
- После выравнивания не появляется regression по содержимому и действиям.

## Обязательные дополнительные условия

- В финальном ответе подготовить `Резюме для коммита` по формату из `ai-context/baseline/guides/tasks.md`.
- После file changes создать отдельный changelog-файл в `ai-context/changelog` и включить в него пользовательский промт, который привёл к изменению.
