# Задача [01-017]

## Короткое описание

Исправить автозакрытие диалогов по неактивности, оповещения об auto-close и
битую кодировку в runtime-логах.

## Цель

Убрать ложное автозакрытие активных диалогов, вернуть операторские
уведомления при автоматическом закрытии по отсутствию активности и поправить
битую кодировку в файлах, которые попадают в startup/runtime-логи.

## Контекст

- Автозакрытие по таймауту выполняет `java-bot`, а не `spring-panel`.
- Операторские ответы в panel пишутся в `chat_history` и обновляют
  `ticket_active`, но для надёжности автозакрытие должно смотреть на реальную
  историю активности, а не только на одну таблицу.
- В `spring-panel` сейчас нет отдельного уведомления операторам о системном
  auto-close по причине `inactivity`.
- В логах и части файлов уже видны следы mojibake, в том числе вокруг строк
  `Авто-система`.

## Что сделать

- Исправить критерий автозакрытия в `java-bot`, чтобы учитывалась фактическая
  последняя активность диалога.
- Нормализовать системную причину auto-close и связанную строку `resolved_by`.
- Добавить в `spring-panel` уведомление операторам о закрытии диалога по
  отсутствию активности.
- Поправить кодировку в runtime-значимых файлах/ресурсах, которые дают битые
  строки в логах.

## Критерии готовности

- Диалог не закрывается автоматически, если после порога бездействия в нём была
  фактическая активность оператора или клиента.
- При автоматическом закрытии по неактивности операторы получают уведомление в
  panel.
- В startup/runtime-логах не появляются битые строки из исправленных файлов.

## Обязательные дополнительные условия

- В финальном ответе подготовить `Резюме для коммита` по формату из
  `ai-context/baseline/guides/tasks.md`.
- После любых file changes создать отдельный changelog-файл в
  `ai-context/changelog` по правилу «одно изменение = один файл».
