# Задача [01-016]

## Короткое описание

Добавить внутреннюю интеграцию Notion для импорта статей в базу знаний.

## Цель

Сделать в проекте безопасное подключение к Notion по integration token без
публичного открытия базы, чтобы можно было импортировать статьи выбранных
авторов в локальный раздел «База знаний».

## Контекст

- Пользователь не хочет публиковать базу Notion наружу и предпочитает
  авторизацию в реальном времени.
- В проекте уже есть локальная база знаний с отдельной сущностью статей,
  редактором и списком материалов.
- Общие настройки системы сохраняются через `settings.json` и страницу
  `/settings`, поэтому конфигурацию интеграции удобно хранить там же.
- Для повторного импорта нужен upsert по внешнему источнику, чтобы статьи не
  плодили дубликаты.
- Целевые авторы для импорта: `Синицын Владимир`, `Лемешкин Андрей`, `Egor`,
  `Канделаки Данил`, `Шумкова Дарья`.

## Что сделать

- Добавить в проект конфигурацию подключения к Notion и UI для её сохранения.
- Реализовать backend-сервис подключения к Notion и импорт статей по списку
  авторов в markdown-формате.
- Доработать локальную базу знаний для сохранения метаданных внешнего
  источника и повторного обновления статей без дублей.
- Добавить действие запуска импорта из интерфейса базы знаний.

## Критерии готовности

- В настройках можно сохранить токен, ссылку на базу или data source, список
  авторов и маппинг полей Notion.
- Из интерфейса можно проверить подключение и запустить импорт статей в
  локальную базу знаний.
- Импортированные статьи сохраняются в markdown и при повторном импорте
  обновляются, а не создаются заново.
