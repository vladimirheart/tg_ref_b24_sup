# Задача [01-104]

Добавить переадресацию незакрытого диалога между пользователями панели, чтобы текущий ответственный мог передать диалог сменщику или другому сотруднику.

## Цель

- дать явный UX для передачи открытого диалога другому пользователю панели;
- обновлять primary responsible без ручных обходных сценариев;
- сохранять прозрачность для уведомлений и списка диалогов.

## Контекст

- сейчас primary owner хранится в `ticket_responsibles.responsible`, а quick action поддерживает только сценарий `take`;
- для передачи сменщику нужен выбор target user и обновление ответственного в runtime/UI;
- логика переадресации должна сочетаться с участниками диалога, а не конфликтовать с ними.

## Что сделать

- добавить backend action для reassignment / redirect конкретного диалога на выбранного пользователя;
- использовать общий user-picker/participant context, чтобы не плодить дублирующий UI;
- обновить dialog list, dialog details и workspace meta после успешной передачи;
- отправлять уведомления участникам и новому ответственному о переадресации.

## Критерии готовности

- открытый незакрытый диалог можно передать от одного пользователя другому;
- после передачи везде обновляется текущий ответственный;
- уведомления и UI не оставляют пользователя в ambiguous состоянии, кто теперь owner.

## Дополнительные условия

- после file changes создать отдельный changelog в `ai-context/changelog`;
- в финальном ответе подготовить `Резюме для коммита`.
