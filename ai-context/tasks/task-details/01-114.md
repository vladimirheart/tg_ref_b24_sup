# Задача [01-114]

## Короткое описание

Унифицировать открытие дочерних модалок на странице настроек по общему
child/workspace-паттерну.

## Цель

Привести дочерние окна settings к одному runtime-паттерну: predictable
workspace/child modal behavior, единые размеры, sheet-expansion там, где это
нужно, и совместимость с parent modal shell.

## Контекст

- В settings используются разные child-модалки:
  обычные `settings-child-modal`, расширенные
  `settings-child-modal--workspace`, а также `settings-child-modal--sheet`.
- Без общего паттерна дочерние окна начинают расходиться по размерам,
  поведению, stacking и способу открытия.
- Эта задача считается runtime/UX baseline для `01-128` и `01-129`.

## Что сделать

- Выделить общий child/workspace modal pattern для дочерних settings-окон.
- Привести размеры, scroll behavior, sheet-expansion и header actions к
  единым правилам.
- Синхронизировать открытие дочерних форм из родительских модалок так, чтобы
  они вели себя одинаково независимо от конкретного subdomain.

## Критерии готовности

- Дочерние settings-модалки используют единый pattern открытия и поведения.
- Workspace/sheet-модификаторы работают предсказуемо и единообразно.
- Вложенные формы не ломают parent modal shell и не расходятся по UX.

## Обязательные дополнительные условия

- В финальном ответе подготовить `Резюме для коммита` по формату из `ai-context/baseline/guides/tasks.md`.
- После file changes создать отдельный changelog-файл в `ai-context/changelog` и включить в него пользовательский промт, который привёл к изменению.
