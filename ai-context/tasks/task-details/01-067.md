# Задача [01-067]

Починить приём клиентской оценки после закрытия диалога в MAX-боте.

## Цель

Сделать так, чтобы MAX-бот корректно принимал числовую оценку по завершённому обращению, сохранял её в `feedbacks`, закрывал pending-запрос и отправлял клиенту подтверждение вместо повторного опроса каждые 2 минуты.

## Контекст

- Повторный запрос оценки уходит из `EngagementTasks.dispatchPendingFeedbackRequests()` по cron раз в 2 минуты, пока в `pending_feedback_requests` остаётся активная запись без обработанной оценки.
- В Telegram-боте уже есть отдельная ветка `tryHandleFeedback(...)`, которая находит pending-запрос, валидирует оценку и вызывает `FeedbackService.storeFeedback(...)`.
- В `MaxWebhookController` аналогичной ветки сейчас нет, поэтому ответ клиента с оценкой не снимает pending-feedback сценарий.
- После любых file changes нужно оставить отдельную запись в `ai-context/changelog`.

## Что сделать

- Добавить в MAX-бот обработку входящей оценки по аналогии с Telegram: поиск активного `PendingFeedbackRequest`, валидация рейтинга, сохранение `feedback` и ответ клиенту.
- Подключить `FeedbackService` в `MaxWebhookController` и не ломать текущий flow диалогов, отмены и public-form continuation.
- Обновить тесты `MaxWebhookController`, чтобы зафиксировать корректный приём оценки и ответ клиенту.

## Критерии готовности

- Ответ клиента числом `1..N` на запрос оценки в MAX сохраняется через `FeedbackService`, а pending-запрос больше не переотправляется как новый.
- Клиент получает текст подтверждения после сохранения оценки.
- Автотест покрывает хотя бы базовый happy-path обработки оценки в MAX.

## Обязательные дополнительные условия

- После любых file changes создать отдельный changelog-файл в `ai-context/changelog`.
- В финальном ответе подготовить `Резюме для коммита`.
