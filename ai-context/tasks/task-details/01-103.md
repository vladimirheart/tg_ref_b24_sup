# Задача [01-103]

Добавить возможность подключать к открытому диалогу других пользователей панели.

## Цель

- дать оператору возможность явно добавить коллег к рабочему диалогу;
- показывать этих пользователей в UI диалога;
- включить их в адресатов уведомлений и контекст работы по диалогу.

## Контекст

- сейчас диалог имеет одного `responsible` в `ticket_responsibles`, а уведомления дополнительно учитывают `ticket_active`;
- на UI есть только self-assign (`Назначить мне` / `Взять себе`);
- в системе уже есть API списка пользователей панели и notification routing по identity.

## Что сделать

- спроектировать хранение участников диалога отдельно от primary responsible;
- добавить backend API для чтения/добавления/удаления участников;
- добавить в dialog modal UI для выбора пользователей панели и отображения списка подключённых участников;
- включить участников в notification recipients и диалоговый контекст.

## Критерии готовности

- из открытого диалога можно добавить другого пользователя панели;
- добавленный пользователь виден в UI как участник диалога;
- участники получают уведомления по диалогу без смены primary responsible.

## Дополнительные условия

- после file changes создать отдельный changelog в `ai-context/changelog`;
- в финальном ответе подготовить `Резюме для коммита`.
