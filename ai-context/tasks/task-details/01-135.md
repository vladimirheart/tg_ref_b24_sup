# Задача [01-135]

## Короткое описание

Дочистить server-rendered payload/data contract страницы `settings` после
runtime split.

## Цель

- сузить `settingsPageInitPayload` в `settings/index.html` до действительно
  нужного runtime contract;
- убрать из initial HTML лишние legacy/dedicated data blocks, которые уже не
  обязаны ехать в общем page-level payload;
- завершить финальный template-side cleanup отдельно от уже выполненного
  client-side runtime split из `01-129`.

## Контекст

- задача `01-129` к этому моменту уже довела основной client-side scope:
  giant settings runtime разрезан, page shell/bootstrap/init вынесены, а
  callback/runtime compatibility-слои в основном сняты;
- в `settings/index.html` остался отдельный потенциальный cleanup-хвост не по
  JS-архитектуре, а по самим server-rendered данным, которые попадают в
  `settingsPageInitPayload`;
- этот шаг не считается обязательным blocker'ом для завершения `01-129`, но
  полезен как отдельный follow-up для упрощения template contract и снижения
  лишних зависимостей между Thymeleaf payload и runtime.

## Что проверить

- нет ли дублей между соседними payload-ключами внутри `dialog`, `channels`,
  `parameters`, `appearance`, `locations`, `admin`;
- не остались ли слишком широкие subdomain bundles, которые тащат данные
  “на всякий случай”, а не под реальный runtime API;
- какие server-injected data blocks можно сузить или вынести из общего initial
  payload без потери текущих сценариев settings page;
- можно ли разделить editable state и справочники/метаданные внутри payload
  более явно, не возвращая giant inline-orchestration.

## Ожидаемые кандидаты

- `dialog.config` рядом с `autoCloseConfig/autoCloseFallbackHours/fallbackCategories`;
- `channels.integrationNetwork` и `channels.integrationNetworkProfiles`;
- `parameters.networkProfiles`, `parameterDependencies`,
  `contractUsageData`, `itConnectionCategoryFields`,
  `itConnectionFallbackLabels`;
- `appearance.statusUsage`;
- `admin.managerLocationBindings`;
- `locations.tree`, `locations.iikoServerSources`,
  `locations.iikoSyncSettings`.

## Критерии готовности

- `settingsPageInitPayload` не содержит очевидных дублей или payload-хвостов,
  оставшихся только по исторической инерции;
- итоговый template contract в `settings/index.html` остаётся рабочим для
  всех текущих сценариев settings page;
- cleanup оформлен как отдельный пакет, а не дописывается задним числом в
  уже завершённый scope `01-129`.
