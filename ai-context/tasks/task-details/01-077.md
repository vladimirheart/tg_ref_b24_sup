# Задача [01-077]

Показать понятный результат ручной/плановой синхронизации департаментов и не терять локации, которые ушли из активных в iiko.

## Цель

- после успешной sync в UI видно, что именно произошло;
- если департамент раньше был активен, а затем исчез из активных департаментов iiko, он не удаляется из структуры;
- такая локация остаётся в дереве `Структура локаций`, но получает статус `Закрыт`;
- в вопросы клиентов продолжают попадать только активные локации.

## Контекст

- Сейчас блок синхронизации показывает прогресс и ошибки, но при успехе не даёт содержательного итога.
- Live-каталог строится только из активных департаментов iiko, поэтому ранее известные локации пропадают из структуры целиком.
- В проекте уже есть карта `statuses`, и `SettingsCatalogService` умеет исключать `Закрыт` из каскадных списков вопросов.

## Что сделать

- В live-сборке каталога объединять актуальные активные департаменты с уже сохранённой структурой.
- Для локаций, которые были раньше, но больше не пришли из iiko как активные, выставлять статус `Закрыт`.
- Для вернувшихся обратно активных локаций выставлять статус `Активен`.
- В `IikoDepartmentLocationsSyncService` сформировать детализированный результат синхронизации:
  - сколько активных локаций всего;
  - сколько закрытых локаций всего;
  - сколько добавлено;
  - сколько закрыто этой синхронизацией;
  - сколько переоткрыто;
  - при необходимости короткие списки изменённых локаций.
- Отобразить результат в UI блока `Синхронизация iikoServer`.
- Обновить тесты на сохранение закрытых локаций и на результат успешной sync.

## Критерии готовности

- После успешной sync в UI виден не только общий статус, но и итог изменений.
- Локация, исчезнувшая из активных департаментов iiko, остаётся в дереве и получает статус `Закрыт`.
- Закрытые локации не попадают в каскадные списки вопросов клиентов.
