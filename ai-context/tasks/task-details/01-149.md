# Задача [01-149]

## Короткое описание

Пробить кэш dialogs assets и восстановить HTML composer модалки диалога.

## Цель

Устранить ситуацию, когда пользователь продолжает видеть старое поведение dialogs-модалки после правок в JS, и вернуть нормальные подписи/placeholder в HTML composer модалки.

## Контекст

- В `dialogs/index.html` dialogs runtime подключаются через статический `dialogsAssetVersion`.
- До правки версия была захардкожена как `20260703-1`, из-за чего браузер мог продолжать держать старые JS-файлы.
- Пользователь подтвердил, что после предыдущих изменений поведение не изменилось.

## Что сделать

- Обновить `dialogsAssetVersion`, чтобы браузер гарантированно запросил новую версию dialogs runtime.
- Восстановить текстовые подписи composer-блока модалки диалога в шаблоне `dialogs/index.html`.
- Зафиксировать follow-up в append-only changelog.

## Критерии готовности

- Страница dialogs получает новый query-version для runtime-скриптов.
- Composer модалки рендерится с нормальными подписями и placeholder.

## Обязательные дополнительные условия

- В финальном ответе подготовить `Резюме для коммита`.
- После file changes создать отдельный changelog-файл в `ai-context/changelog`.
