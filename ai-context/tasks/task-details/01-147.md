# Задача [01-147]

## Короткое описание

Починить отправку вложений из модалки диалога и восстановить читаемую кодировку в modal runtime.

## Цель

Убрать ситуацию, когда выбранные вложения отображаются в composer модалки, но при ошибке отправки фактически пропадают и не уходят клиенту, а также восстановить нормальный русский текст в элементах модалки диалога.

## Контекст

- Отправка медиа из модалки идёт через `sendMediaFiles()` в `dialogs-details-history-runtime.js`.
- Сейчас media flow очищает pending-вложения в `finally`, даже если `/media` запрос завершился ошибкой.
- В модальных runtime-файлах были участки с битой кодировкой, из-за чего часть подписей и сообщений отображалась некорректно.

## Что сделать

- Сохранять pending-вложения при неуспешной отправке медиа, чтобы оператор мог повторить отправку.
- Очищать staged/select-вложения только после успешного media send.
- Восстановить нормальную кодировку пользовательских строк в runtime модалки диалога.
- Проверить синтаксис изменённых JS-файлов.
- Зафиксировать результат в append-only changelog.

## Критерии готовности

- Вложения из модалки отправляются по прежнему `/media` flow.
- Если отправка падает, выбранные вложения не теряются автоматически.
- В модалке больше нет видимых участков с битой кодировкой в затронутых runtime.
- Изменённые JS-файлы проходят синтаксическую проверку.

## Обязательные дополнительные условия

- В финальном ответе подготовить `Резюме для коммита`.
- После file changes создать отдельный changelog-файл в `ai-context/changelog`.
