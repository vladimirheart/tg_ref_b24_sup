# Задача [01-006]

## Короткое описание

Добавить для VLESS в настройках явные поля `host/port/token` и валидацию `token`.

## Цель

Сделать конфигурацию VLESS прозрачной и безопасной: для этого протокола пользователь должен задавать только релевантные поля.

## Что сделать

- Обновить формы сетевых маршрутов (проект, боты, канал, профиль): для схемы `vless` показывать `token`.
- Сохранить обратную совместимость данных через нормализацию.
- Добавить проверки перед сохранением: для `vless` token обязателен.
- Передать `token` в backend-модель маршрута.

## Критерии готовности

- В UI при выборе `vless` видны `host`, `port`, `token`.
- Маршрут/профиль не сохраняется без `token`.
- Backend принимает/сохраняет `token` и формирует proxy URL для vless.
