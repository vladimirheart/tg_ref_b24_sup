# Задача [01-093]

Починить сохранение паспорта объекта после перевода полей на effective-каталог: убрать `403 Forbidden (CSRF or access denied)` и исправить падение Bootstrap dropdown на кастомных searchable-select.

## Цель

- вернуть рабочее сохранение паспорта через `POST /api/object_passports`;
- убрать JS-ошибку `this._menu is null` при открытии кастомных селектов;
- сохранить существующую логику searchable-select без регрессии в UI.

## Контекст

- страница `passports/new.html` использует `fetch` для сохранения и добавления параметров, но не подключала `common.js` и не рендерила `meta[name="_csrf"]`;
- в проекте CSRF для fetch-запросов централизованно добавляется через `common.js`;
- кастомные селекты страницы помечены `data-force-custom-dropdown`, но на кнопках оставался `data-bs-toggle="dropdown"`, из-за чего Bootstrap data-api всё равно пытался открывать dropdown и падал на отсутствии `_menu`.

## Что сделать

- добавить на страницу паспорта CSRF meta-теги;
- подключить `common.js` до inline-скрипта страницы;
- при custom-dropdown-режиме убирать bootstrap data-api атрибуты у toggle-кнопки;
- расширить smoke-тест страницы проверкой подключения `common.js` и CSRF-meta.

## Критерии готовности

- сохранение паспорта больше не получает `403` из-за отсутствующего CSRF;
- кастомные селекты не роняют `dropdown.js`;
- `ManagementControllerWebMvcTest` проходит;
- компиляция `spring-panel` проходит.

## Дополнительные условия

- после file changes создать отдельный changelog в `ai-context/changelog`;
- в финальном ответе подготовить `Резюме для коммита`.
