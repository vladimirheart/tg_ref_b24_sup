# Задача [01-148]

## Короткое описание

Дожать paste/staging вложений в модалке диалога и убрать хрупкую привязку к `instanceof HTMLInputElement`.

## Цель

Убрать ситуацию, когда при вставке скриншота из буфера оператор получает warning про невозможность автоматического прикрепления, хотя сам clipboard media flow должен уметь staging в скрытый `file` input.

## Контекст

- Вставка скриншота в модалке идёт через `extractClipboardImageFiles()` и `stageMediaFilesInInput()`.
- Ранее helper’ы media input опирались на `instanceof HTMLInputElement`, что делает flow хрупким для нестандартного окружения/realm и может возвращать `0` без фактического staging.
- Пользователь подтвердил, что по paste flow всё ещё видит warning: `Не удалось прикрепить скриншот автоматически. Используйте кнопку прикрепления медиа.`

## Что сделать

- Ослабить проверку media input до duck-typing по `type === 'file'` и наличию `files`.
- Использовать эту же проверку в чтении selected files, staged files и очистке pending media.
- Проверить синтаксис затронутых JS-файлов.
- Зафиксировать follow-up в append-only changelog.

## Критерии готовности

- Paste flow больше не ломается из-за жёсткой проверки `instanceof HTMLInputElement`.
- Staging/reading/clearing pending media используют единый helper определения media input.
- Изменённые JS-файлы проходят синтаксическую проверку.

## Обязательные дополнительные условия

- В финальном ответе подготовить `Резюме для коммита`.
- После file changes создать отдельный changelog-файл в `ai-context/changelog`.
