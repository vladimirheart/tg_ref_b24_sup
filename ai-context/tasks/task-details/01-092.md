# Задача [01-092]

Перевести поля `Департамент`, `Страна`, `Бизнес`, `Юридическое лицо`, `Тип партнёра`, `Город` на странице паспорта объекта на корректные источники со страницы настроек, чтобы location-поля брались из effective-каталога и live-структуры `iiko` по аналогии с блоком `Структура локаций`.

## Цель

- убрать расхождение между страницей паспорта объекта и страницей настроек;
- сделать `business / partner_type / country / city / department` зависимыми от того же effective locations catalog, что и `Структура локаций`;
- сохранить `legal_entity` связанным с настройками сети и профилями, не ломая текущий UX паспорта.

## Контекст

- сейчас `ManagementController.populatePassportEditor()` собирает паспортный payload в основном из `settingsParameterService.listParameters(false)`, из-за чего страница паспорта может видеть устаревшие или неполные данные;
- страница настроек уже использует `IikoDepartmentLocationCatalogService.loadCatalog()` и `buildEffectiveLocationsPayload(...)`, где effective-дерево дополняется live-структурой из `iikoServer API`;
- в `passports/new.html` каскадная фильтрация полей опирается на `parameterValuesPayload`, поэтому для исправления достаточно отдать туда правильный backend payload с зависимостями;
- ориентир по источнику данных: `Настройки -> Структура локаций`.

## Что сделать

- загрузить для паспорта effective locations payload через `IikoDepartmentLocationCatalogService`;
- на его основе пересобрать `country / partner_type / business / city / department` в passport parameter payload с корректными dependency-связями;
- оставить `legal_entity` добираемым из настроек сети и network profiles;
- добавить WebMvc regression test, подтверждающий, что редактор паспорта видит live-локации из effective-каталога.

## Критерии готовности

- в редакторе паспорта значения `Бизнес`, `Тип партнёра`, `Страна`, `Город`, `Департамент` совпадают по источнику с effective-каталогом настроек;
- значения из live-структуры `iiko` отображаются в паспорте без ручной синхронизации `settings_parameters`;
- сборка `spring-panel` проходит;
- таргетный `ManagementControllerWebMvcTest` проходит.
