# Задача [01-050]

## Короткое описание

Добавить оперативный просмотр состава лицензий по каждой RMS.

## Цель

Дать оператору возможность быстро открыть список лицензий, доступных для
конкретной RMS-записи, без ручного разбора сырого ответа RMS.

## Контекст

Сейчас RMS-мониторинг показывает только итоговое состояние лицензии и дату
истечения ближайшей из них. Пользователю иногда нужно видеть именно состав
лицензий по конкретной RMS. Удобнее всего показывать последние сохранённые
данные в модальном окне из действий по строке.

## Что сделать

- Сохранять детализацию лицензий RMS после обновления лицензии.
- Добавить backend endpoint для получения сохранённого списка лицензий по RMS.
- Добавить на страницу `/analytics/rms-control` действие просмотра лицензий по записи.
- Показать данные в быстром модальном окне с последним временем проверки.

## Критерии готовности

- У записи RMS есть действие просмотра лицензий.
- Список лицензий можно открыть без отдельной страницы.
- Если детализация ещё не получена, UI показывает понятное сообщение.
- `spring-panel\\mvnw.cmd -q -DskipTests compile` проходит успешно.

## Обязательные дополнительные условия

- В финальном ответе подготовить `Резюме для коммита` по формату из
  `ai-context/baseline/guides/tasks.md`.
- После любых file changes создать отдельный changelog-файл в
  `ai-context/changelog` по правилу «одно изменение = один файл».
