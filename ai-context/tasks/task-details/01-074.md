# Задача [01-074]

Добавить в настройки локаций ручную синхронизацию департаментов из `iikoServer API` с видимым прогрессом, вынести источники в отдельную вкладку и дать пользователю настраивать расписание синхронизации.

## Цель

Сделать управление live-синхронизацией локаций полноценным:

- запуск вручную без рестарта;
- понятный статус выполнения в UI;
- редактирование источников отдельно от дерева локаций;
- настраиваемый интервал автоматической синхронизации.

## Контекст

- Сейчас `iikoServer`-источники уже перенесены в `Настройки -> Структура локаций`, но они находятся в одной вкладке с самим деревом.
- Плановая синхронизация работает только scheduler'ом по фиксированному интервалу.
- Пользователь запросил принудительный запуск и status/progress bar.

## Что сделать

- Вынести настройки источников `iikoServer API` в отдельную вкладку внутри модалки локаций.
- Добавить отдельный блок управления sync:
  - кнопка `Синхронизировать сейчас`;
  - progress/status bar;
  - текущий статус последней синхронизации.
- Добавить настройку расписания:
  - включена/выключена автосинхронизация;
  - интервал в минутах.
- Перевести scheduler на чтение runtime-настроек расписания из shared settings.
- Добавить API для ручного запуска и получения статуса sync.

## Критерии готовности

- Пользователь может запустить sync вручную из UI.
- Во время sync в UI виден прогресс и статус.
- Источники `iikoServer` редактируются в отдельной вкладке, а дерево локаций остаётся в своей.
- Интервал автосинхронизации задаётся из настроек и используется scheduler'ом без жёстко зашитого фиксированного значения.
