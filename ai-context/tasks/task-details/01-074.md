# Задача [01-074]

## Короткое описание

Добавить ручную sync локаций с прогрессом и настраиваемым расписанием.

## Цель

Сделать синхронизацию департаментов из `iikoServer` управляемым пользовательским сценарием:
оператор должен видеть возможность запустить sync вручную, понимать прогресс, и отдельно
настраивать его расписание.

## Контекст

- На странице настроек уже присутствует отдельная вкладка синхронизации `iikoServer`, а в runtime
  видны контуры `/api/settings/locations-sync/status` и `/api/settings/locations-sync/run`.
- В backend уже есть выделенные слои `SettingsLocationsSyncController`,
  `IikoDepartmentLocationsSyncService`, `LocationsIikoSyncSettingsService` и scheduler.
- Для `Track A3` это важный baseline не только по данным, но и по operator UX: location sync
  нельзя оставлять “невидимой фоновый магией”.
- Задача должна сохранить bounded semantics sync-контура и не растворить её в общем giant
  settings runtime.

## Пересечения с backlog

- `01-071`..`01-073` задают корректный источник и save-модель; `01-074` делает sync
  операционно управляемой.
- `01-075` влияет на успешность ручного sync, а `01-077` — на прозрачность его результата.
- `01-128` / `01-129` при будущем split обязаны сохранить status/progress/run flow без регрессии.

## Что сделать

- Добавить на страницу настроек ручной запуск sync департаментов.
- Показывать оператору прогресс, текущий этап и итоговый статус синхронизации.
- Дать возможность управлять расписанием автосинхронизации отдельно от ручного запуска.
- Сохранить совместимость между UI, scheduler и status endpoint.

## Критерии готовности

- Оператор может запустить sync вручную со страницы настроек.
- Во время выполнения sync виден прогресс и понятные стадии выполнения.
- Расписание автосинхронизации настраивается явно и не мешает ручному запуску.
- Ручной и плановый sync используют один и тот же bounded backend flow.

## Обязательные дополнительные условия

- В финальном ответе подготовить `Резюме для коммита` по формату из
  `ai-context/baseline/guides/tasks.md`.
- После file changes создать отдельный changelog-файл в `ai-context/changelog` и включить в него
  пользовательский промпт, который привёл к изменению.
