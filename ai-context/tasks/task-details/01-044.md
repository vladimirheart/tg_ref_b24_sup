# Задача [01-044]

## Короткое описание

Добавить в раздел аналитики новую страницу рядом с "Мониторинг SSL" для контроля RMS: адрес, имя сервера, дата окончания лицензии, статус лицензии, статус доступности RMS, тип сервера, хранение кредов и автоматические/ручные обновления через очереди.

## Что нужно сделать

- Добавить отдельную страницу аналитики с таблицей RMS-узлов.
- Хранить для каждой записи адрес, логин, пароль и флаг включения мониторинга.
- Получать имя сервера, тип и дату окончания лицензии через RMS REST API по заметкам из `clear_bat_analysis.md`.
- Подсвечивать статус лицензии:
  - больше 7 дней: `ok`
  - меньше или равно 7 дням: `warning`
  - меньше или равно 3 дням: `critical`
- При остатке лицензии меньше недели отправлять уведомление операторам.
- Обновлять лицензии автоматически 1 раз в сутки с паузой 20 секунд между записями.
- Проверять доступность RMS `ping`-ом из 4 пакетов каждые 5 минут с паузой 20 секунд между завершёнными проверками.
- При недоступности строить трассировку, сохранять краткий итог и давать возможность скачать полный результат.
- Добавить ручной запуск обновления лицензий и сетевых статусов с той же очередной логикой.

## Ограничения

- Новые запуски авто- и ручных обновлений не должны дублировать параллельные очереди.
- Пароль нельзя возвращать на клиент в открытом виде.
- Для выключенной записи автообновления не должны выполняться.
