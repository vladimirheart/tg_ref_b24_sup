# Задача [01-144]

## Короткое описание

Восстановить дерево локаций в настройках после sync iiko и пересчитать read-логику bell-уведомлений.

## Цель

Убрать регрессию, при которой вкладка `Структура локаций` на странице настроек остаётся пустой даже после успешной синхронизации с `iiko`, и перевести bell-уведомления на более предсказуемую UX-логику: видимые записи становятся прочитанными через 2 секунды после открытия dropdown, плюс появляется явное действие `Прочитать все`.

## Контекст

- Последние changes по `locations` уже выносили lazy page-data загрузку и нормализовали `locations.json`, поэтому новый фикс должен учитывать клиентский cache и live refresh path.
- Источник effective locations используется не только в settings UI, но и в bot question flow через общий `config/shared/locations.json`, поэтому нужно отдельно проверить, что выбор бизнеса/города/локации для клиента не ломается.
- Текущая логика bell-уведомлений помечает запись прочитанной только по клику в конкретное уведомление; пользователь просит read-on-visible с задержкой 2 секунды и явный bulk action.
- Изменения затрагивают и frontend runtime, и notification API contract, поэтому нужен regression-pass по settings, notifications и bot settings.

## Что сделать

- Найти и устранить причину, по которой `Структура локаций` может оставаться пустой после успешной `iiko` sync, с учётом page-data cache/runtime refresh.
- Обновить bell dropdown так, чтобы после открытия через 2 секунды помечались прочитанными только те уведомления, которые реально видны в списке.
- Добавить явную кнопку `Прочитать все` и сохранить корректный unread badge / rearm semantics.
- Прогнать смежные тесты, подтверждающие, что общий locations payload не ломает клиентские вопросы боту.

## Критерии готовности

- После успешной `iiko` sync и/или повторного открытия модалки settings дерево локаций показывает актуальные записи, а не пустой state.
- Видимые уведомления в bell dropdown автоматически становятся прочитанными только после 2 секунд видимости.
- В dropdown есть явное действие `Прочитать все`, которое не ломает следующий rearm новых уведомлений.
- Regression-проверка по `locations` и `question_flow` бота проходит.

  пользовательский промпт, который привёл к изменению.
