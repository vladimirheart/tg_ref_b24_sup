# Задача [01-120]

## Короткое описание

Добавить на dashboard метрику и heatmap нагрузки по часам.

## Цель

Показать на dashboard интенсивность обращений по часам так, чтобы сверху была краткая числовая метрика, а ниже — наглядная раскладка по дням недели и часам.

## Контекст

- Пользователь запросил `нагрузку по часам`, уточнив, что это количество обращений в час, и приложил пример в виде day/hour heatmap.
- В `DashboardAnalyticsService` уже есть доступ к дате и времени создания обращения через `createdAt`, `createdDate`, `createdTime`.
- На dashboard уже есть KPI-блок и аналитическая сетка; там можно встроить новый KPI и отдельный широкий блок под heatmap.

## Что сделать

- Добавить в payload dashboard данные по средней нагрузке в час, пиковому часу и матрице `день недели x час`.
- Заменить декоративную KPI-карточку на реальную метрику нагрузки в час.
- Добавить в dashboard новый широкий heatmap-блок по дням недели и часам.

## Критерии готовности

- На dashboard появляется числовая метрика нагрузки в час.
- На dashboard появляется тепловая карта по дням недели и часам на основе текущего фильтра.
- Данные метрики и heatmap обновляются вместе с остальными фильтрами dashboard.

## Обязательные дополнительные условия

- В финальном ответе подготовить `Резюме для коммита` по формату из `ai-context/baseline/guides/tasks.md`.
- После file changes создать отдельный changelog-файл в `ai-context/changelog` и включить в него пользовательский промт, который привёл к изменению.
