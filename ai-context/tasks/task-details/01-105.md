# Задача [01-105]

Починить сохранение нового паспорта объекта в UI: после нажатия `Готово` и затем `Сохранить` страница показывает предупреждение о попытке уйти со страницы и мешает завершить сценарий.

## Цель

- убрать ложный `beforeunload` при успешном сохранении нового паспорта;
- сохранить защиту от потери несохранённых изменений для реального ухода со страницы;
- не менять backend API и остальной сценарий редактирования существующего паспорта.

## Контекст

- страница `passports/new.html` использует два независимых сторожа несохранённых изменений:
  - локальный `hasPassportUnsavedChanges` + `beforeunload`;
  - общий `window.EditGuard`;
- при создании нового паспорта `savePassport()` после успешного `POST /api/object_passports` делает redirect на `/object-passports/{id}`;
- до редиректа dirty-состояние не сбрасывалось, поэтому браузер воспринимал программный переход после сохранения как попытку уйти со страницы с несохранёнными правками.

## Что сделать

- добавить единый helper сброса dirty-состояния после успешного сохранения;
- вызывать его перед redirect для нового паспорта и после обычного успешного сохранения;
- зафиксировать изменение отдельным changelog-файлом с пользовательским промтом.

## Критерии готовности

- после успешного сохранения нового паспорта переход на `/object-passports/{id}` не вызывает ложное предупреждение браузера;
- существующий guard продолжает предупреждать только о реально несохранённых изменениях;
- финальный ответ содержит `Резюме для коммита`.
