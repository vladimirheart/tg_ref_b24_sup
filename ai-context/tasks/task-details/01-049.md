# Задача [01-049]

## Короткое описание

Добавить точечные refresh-действия и массовые переключатели функционала для RMS.

## Цель

Дать оператору возможность отдельно по каждой RMS-записи запускать проверку
лицензии и доступности, а также массово включать и отключать оба типа
мониторинга независимо друг от друга.

## Контекст

Сейчас у RMS есть только один общий флаг `enabled`, а ручное обновление есть
только массовое. Пользователь запросил:

- точечный запуск проверки лицензии по записи;
- точечный запуск проверки доступности по записи;
- массовое включение/отключение мониторинга лицензий;
- массовое включение/отключение мониторинга доступности.

Для этого нужно разделить один общий флаг на отдельные фичи уровня RMS-записи:
мониторинг лицензии и мониторинг доступности.

## Что сделать

- Добавить в RMS-модель и хранилище отдельные флаги:
  `license_monitoring_enabled` и `network_monitoring_enabled`.
- Расширить backend API точечными refresh-операциями по записи.
- Добавить backend API массового включения/отключения обоих видов мониторинга.
- Обновить страницу `/analytics/rms-control` и `rms-monitoring.js`:
  показать отдельные состояния и добавить удобные действия по записи.

## Критерии готовности

- По каждой RMS-записи можно отдельно запросить проверку лицензии.
- По каждой RMS-записи можно отдельно запросить проверку доступности.
- Мониторинг лицензий можно массово включать и отключать.
- Мониторинг доступности можно массово включать и отключать.
- `spring-panel\\mvnw.cmd -q -DskipTests compile` проходит успешно.
