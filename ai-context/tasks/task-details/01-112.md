# Задача [01-112]

## Короткое описание

Уплотнить UI блока «Каналы (боты)» в настройках и спрятать шумные сетевые
секции за явными действиями.

## Цель

Сделать основной каналовый workflow компактным: управление ботами должно быть
в центре, а сетевые маршруты, proxy/VPN profiles и соседние технические секции
не должны перегружать первый экран.

## Контекст

- Модалка `Каналы (боты)` в `settings/index.html` содержит основной manage-flow,
  шаблоны, automation и network-adjacent sections.
- Пользовательский сценарий в первую очередь про добавление/редактирование
  канала, а не про постоянный просмотр route/proxy профилей.
- Эта задача является UX baseline для `01-128`, `01-129` и `01-131`: новый
  split не должен возвращать шумные сетевые блоки в основной channel shell.

## Что сделать

- Уплотнить верхний manage-блок `Каналы (боты)` и оставить в фокусе работу с
  каналами.
- Скрыть сетевые/proxy/VPN sections за отдельными явными действиями
  `Открыть список`, `Настроить сеть`, `Новый профиль` или аналогичным UX.
- Сохранить доступность route/profile настроек без перегрузки основного канала.

## Критерии готовности

- Основной экран `Каналы (боты)` в первую очередь показывает управление
  каналами и статусами, а не сеть.
- Proxy/VPN/network sections доступны, но не доминируют в основном потоке.
- Добавление и редактирование канала остаётся понятным и быстрым.

## Обязательные дополнительные условия

- В финальном ответе подготовить `Резюме для коммита` по формату из `ai-context/baseline/guides/tasks.md`.
- После file changes создать отдельный changelog-файл в `ai-context/changelog` и включить в него пользовательский промт, который привёл к изменению.
