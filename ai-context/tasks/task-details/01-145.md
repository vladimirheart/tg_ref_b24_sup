# Задача [01-145]

## Короткое описание

Скорректировать whitelist типов `iiko` corporate items для локаций.

## Цель

Убрать попадание лишних групповых записей в live-каталог локаций и оставить только корректные типы источника: `DEPARTMENT`, `MANUFACTURE`, `CENTRALSTORE`.

## Контекст

- Текущий gateway читает XML `/resto/api/corporation/departments/` и фильтрует элементы по `type`.
- После предыдущего fixes по `locations` выяснилось, что набор допустимых типов нужно уточнить: `GROUP` попадать не должен, а `MANUFACTURE` и `CENTRALSTORE` должны считаться валидными source items наравне с `DEPARTMENT`.
- Изменение точечное, но влияет на live source-of-truth для settings tree и downstream consumers effective locations payload.

## Что сделать

- Обновить whitelist типов в `IikoDepartmentLocationCatalogService`.
- Добавить regression-test на XML-ответ, где одновременно присутствуют `DEPARTMENT`, `MANUFACTURE`, `CENTRALSTORE` и лишний `GROUP`.
- Зафиксировать изменение в append-only changelog.

## Критерии готовности

- `GROUP` и другие неподдерживаемые типы не попадают в список live-локаций.
- `DEPARTMENT`, `MANUFACTURE`, `CENTRALSTORE` попадают в список и участвуют в построении каталога.
- Regression-test на gateway проходит.

## Обязательные дополнительные условия

- В финальном ответе подготовить `Резюме для коммита` по формату из
  `ai-context/baseline/guides/tasks.md`.
- После file changes создать отдельный changelog-файл в `ai-context/changelog` и включить в него
  пользовательский промпт, который привёл к изменению.
