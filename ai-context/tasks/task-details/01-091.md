# Задача [01-091]

Показать в legacy-модалке диалога блок `Мои диалоги` для текущего оператора панели: отдельные списки `Неотвеченные` и `В работе`, только по его открытым обращениям, с открытием выбранного диалога в той же модалке.

## Цель

- отдать из backend отдельный payload `my_dialogs` по логину текущего оператора;
- показать в `dialogDetailsModal` только мои незакрытые диалоги;
- разделить их на `Неотвеченные` и `В работе`;
- обновлять блок после polling и основных действий оператора.

## Контекст

- текущая legacy-модалка диалога показывает карточку и историю, но не даёт навигации по собственным открытым диалогам оператора;
- в строках списка используется `data-responsible`, где может оказаться display name, а для этой задачи нужен raw login из `Authentication.getName()`;
- `/api/dialogs` уже возвращает общий список и summary, поэтому расширение payload удобнее делать без нового маршрута.

## Что сделать

- добавить в `DialogLookupReadService` группировку моих активных диалогов по raw login текущего пользователя;
- расширить `DialogListReadService` и `DialogsController`, чтобы initial render и polling отдавали `my_dialogs`;
- передавать в таблицу `data-responsible-raw` и использовать его в JS-проверках принадлежности;
- добавить в `templates/dialogs/index.html` секцию `Мои диалоги` внутри `#dialogDetailsSidebar`;
- в `dialogs.js` завести отдельный state, рендер списка и открытие диалога по клику в той же legacy-модалке;
- обновлять список после `take`, `close`, `reopen`, `reply`, `mark as read/open` и polling refresh.

## Критерии готовности

- оператор видит в модалке только свои открытые диалоги;
- неотвеченные диалоги идут отдельным блоком сверху;
- диалоги в работе идут отдельным блоком ниже;
- `resolved` / `closed` не попадают в список;
- чужие диалоги не попадают в список;
- после закрытия диалог исчезает из списка;
- после нового сообщения клиента диалог появляется в `Неотвеченные`;
- клик по элементу открывает выбранный диалог в текущей модалке.
