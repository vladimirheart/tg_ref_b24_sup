# Задача [01-047]

## Короткое описание

Принудительно выровнять UTF-8 для runtime-логов панели и RMS-команд в Windows.

## Цель

Убрать появление нечитаемых символов в логах панели и в результатах системных
команд мониторинга из-за неявной или системной кодировки.

## Контекст

После запуска `spring-panel` в Windows в логах появились строки с mojibake.
При этом `logback-spring.xml` уже пишет в `UTF-8`, а в `RmsLicenseMonitoringService`
вывод `ping` и `tracert` читался через `Charset.defaultCharset()`. Для самой панели
в `run-windows.bat` не было явной фиксации UTF-8 JVM-аргументами.

## Что сделать

- Зафиксировать `UTF-8` при запуске панели через `run-windows.bat`.
- Убрать использование системной кодировки при чтении вывода команд RMS.
- Проверить сборкой, что изменения не ломают `spring-panel`.

## Критерии готовности

- `run-windows.bat` запускает панель с явными UTF-8 JVM-аргументами.
- `RmsLicenseMonitoringService` не использует `Charset.defaultCharset()`.
- `spring-panel\\mvnw.cmd -q -DskipTests compile` проходит успешно.

## Обязательные дополнительные условия

- В финальном ответе подготовить `Резюме для коммита` по формату из
  `ai-context/baseline/guides/tasks.md`.
- После любых file changes создать отдельный changelog-файл в
  `ai-context/changelog` по правилу «одно изменение = один файл».
