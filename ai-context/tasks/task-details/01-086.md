# Задача [01-086]

Починить сборку `spring-panel`, которая падает на `PublicFormService` из-за вызова несуществующего метода `readDialogConfig()` после недавнего выделения runtime config helper.

## Цель

- вернуть успешную компиляцию `spring-panel`;
- выровнять чтение CAPTCHA-конфига в `PublicFormService` на актуальные helper-методы;
- убедиться, что фикс не тянет лишних архитектурных изменений.

## Контекст

- запуск `.\run-windows.bat` падает на этапе `maven-compiler-plugin`;
- ошибка указывает на `PublicFormService.java:[682,33] cannot find symbol method readDialogConfig()`;
- в этом сервисе уже используются `readDialogConfigString(...)`, `readDialogConfigInt(...)` и другие обёртки над `PublicFormRuntimeConfigService`.

## Что сделать

- заменить оставшийся вызов несуществующего `readDialogConfig()` на актуальный string-helper;
- прогнать компиляцию `spring-panel`, чтобы подтвердить устранение ошибки;
- зафиксировать изменение в `ai-context/changelog`.

## Критерии готовности

- `spring-panel` компилируется без ошибки `cannot find symbol readDialogConfig()`;
- логика shared-secret CAPTCHA продолжает читать значение из runtime dialog config;
- изменение ограничено локальным точечным фиксом.

## Дополнительные условия

- после file changes создать отдельный changelog в `ai-context/changelog`;
- в финальном ответе подготовить `Резюме для коммита`.
