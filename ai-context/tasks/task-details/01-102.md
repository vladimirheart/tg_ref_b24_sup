# Задача [01-102]

Заблокировать прокрутку основной страницы при достижении верхнего или нижнего края истории в открытой legacy-модалке диалога.

## Цель

- убрать scroll bleed из модалки диалога в underlying страницу;
- сохранить нормальную прокрутку внутри истории и sidebar модалки;
- не сломать desktop и touch-сценарии.

## Контекст

- история сообщений рендерится в `#dialogDetailsHistory` внутри `dialogDetailsModal`;
- сейчас при достижении края scroll-контейнера событие продолжает прокрутку страницы под модалкой;
- модалка использует mix bootstrap и fallback modal logic в `dialogs.js`, поэтому защита должна работать и без зависимости только на CSS.

## Что сделать

- найти scrollable-контейнеры внутри `dialogDetailsModal`;
- добавить JS/CSS-защиту от передачи прокрутки на `body` при достижении края;
- отдельно учесть wheel и touch/pointer сценарии;
- проверить, что media preview и category modal поверх dialog modal не ломаются.

## Критерии готовности

- при открытой модалке диалога страница под ней не прокручивается от wheel/touch на краях истории;
- история и sidebar модалки продолжают прокручиваться штатно;
- закрытие модалки возвращает обычное поведение прокрутки страницы.

## Дополнительные условия

- после file changes создать отдельный changelog в `ai-context/changelog`;
- в финальном ответе подготовить `Резюме для коммита`.
