# Задача [01-030]

## Заголовок

Реализовать hybrid retrieval, consistency checks и канонические knowledge units

## Контекст

Token overlap retrieval подходит для MVP, но не обеспечивает стабильное качество
на близких по смыслу формулировках. Нужен гибридный поиск и проверка
согласованности evidence перед auto-reply.

## Что нужно сделать

- внедрить hybrid retrieval:
  keyword/BM25 + semantic retrieval + reranking;
- добавить scope-фильтры (`channel/business/location`) до основного ранжирования;
- реализовать consistency checks:
  минимум 2 подтверждения и детект конфликтующих источников;
- создать `ai_agent_knowledge_unit` и `ai_agent_memory_link` для канонического
  слоя знаний и связки с памятью;
- блокировать auto-reply при evidence conflict (fallback в suggestion/escalation).

## Критерии готовности

- качество top-3 кандидатов выше baseline token-only retrieval;
- источник ответа ссылается на каноническую единицу знания или approved memory;
- конфликтующие evidence не приводят к auto-reply;
- пайплайн retrieval прозрачен и отлаживаем через trace/debug данные.
