# Задача [01-070]

Починить создание Spring bean для `SlaRoutingPolicyService`, чтобы запуск `spring-panel` не падал на `No default constructor found`.

## Цель

Восстановить корректный dependency injection для `SlaRoutingPolicyService`, чтобы Spring выбирал основной конструктор и не пытался инстанцировать сервис как bean без аргументов.

## Контекст

- После обхода падения на `maven-clean-plugin` запуск панели дошёл до инициализации Spring-контекста.
- Новый стоппер: `BeanInstantiationException` для `SlaRoutingPolicyService` с причиной `No default constructor found`.
- В сервисе есть несколько публичных конструкторов, но основной DI-конструктор не отмечен как приоритетный для Spring.

## Что сделать

- Явно пометить основной конструктор `SlaRoutingPolicyService` как `@Autowired`.
- Убедиться, что `run-windows.bat` после предыдущей правки проходит дальше `clean` и теперь не падает на этом bean.

## Критерии готовности

- Spring больше не завершает старт на `SlaRoutingPolicyService`.
- `run-windows.bat` проходит минимум до следующего реального runtime-этапа или до полноценного старта приложения.
