# Задача [01-033]

## Короткое описание

Добавить в панель автоматизацию отключения корпоративных скидок через Bitrix24 и iiko.

## Цель

Поднять в `spring-panel` управляемый runtime-контур, который по кнопке из панели:

- находит подходящие задачи Bitrix24 в проекте увольнения сотрудников;
- извлекает телефон сотрудника из тела задачи;
- выполняет действие в iiko по локально заданным интеграционным настройкам;
- отмечает checklist-пункт в Bitrix24 только при успешном выполнении;
- сохраняет историю запусков, результат по каждой задаче и журнал ошибок.

## Контекст

- Пользователь работает с Bitrix24 по webhook.
- Секреты Bitrix24 и iiko должны храниться в панели как личные данные конкретного пользователя, а не в local-machine файлах.
- iiko-контур должен поддерживать несколько URL-профилей с отдельными кредами и отдельным выбором категорий/кошельков для каждого URL.
- На текущем этапе group/project id Bitrix24 и точные iiko-категории будут определяться уже после подключения.
- Для отбора задач нужны только незакрытые задачи, в которых целевой checklist-пункт еще не отмечен.
- Целевые checklist-пункты:
  - `отключение корпоративной скидки`
  - `Отключение корпоративных скидок`

## Что сделать

- Добавить backend-контур автоматизации в `spring-panel` с персональным user-scoped конфигом секретов в панели.
- Добавить UI в панели для запуска, dry-run, просмотра статуса подключения и истории запусков.
- Подключить Bitrix24 REST API для чтения групп, задач и checklist-пунктов, а также для отметки пункта при успехе.
- Подготовить iiko-контур для URL-зависимого discovery организаций, категорий и кошельков и дальнейшего боевого отключения скидок.
- Сохранить журнал запусков и результаты по задачам в БД панели.

## Критерии готовности

- В панели есть отдельный блок автоматизации с запуском и просмотром истории.
- Секреты не пишутся в versioned файлы и хранятся отдельно для каждого пользователя панели.
- Для iiko поддерживаются отдельные URL-профили с независимыми кредами и выбором категорий/кошельков.
- Dry-run умеет показать, какие Bitrix24-задачи попали в обработку и почему.
- История запусков сохраняется в БД с детализацией по каждой задаче.
- Bitrix24 checklist отмечается только после успешного результата по задаче.

## Обязательные дополнительные условия

- По завершении задачи показать алерт через
  `./ai-context/baseline/scripts/show-completion-alert.sh`, если пользователь отдельно
  не отменил это требование.
- В финальном ответе подготовить `Резюме для коммита` по формату из
  `ai-context/baseline/guides/tasks.md`.
- После любых file changes создать отдельный changelog-файл в
  `ai-context/changelog` по правилу «одно изменение = один файл».
