# Задача [01-131]

## Короткое описание

Разрезать ChannelApiController по transport responsibility.

## Цель

Сделать `ChannelApiController` thin transport wrapper'ом вместо mixed
controller, который одновременно держит CRUD каналов, credential flows,
Telegram diagnostics и channel notifications.

## Контекст

- По audit-срезу `ChannelApiController` держит около `1168` строк.
- Внутри него смешаны `/channels`, `/bot-credentials`,
  `/channels/{id}/test-message`, `/channels/{id}/bot-info`,
  `public-id/regenerate` и `/channel-notifications`.
- Это уже не исходный giant hotspot из `01-024`, а secondary transport layer,
  который стал следующим кандидатом на bounded split после UI runtime работ.
- При этом controller уже участвует в существующем backlog по channel runtime и
  editor flows: `01-108`, `01-112` и `01-121` задают пользовательский и API
  baseline, который transport split не должен менять.

## Пересечения с backlog

- `01-108` — Telegram `base_url` / Bot API mirror semantics остаются текущим
  runtime contract.
- `01-112` — compact channel settings UI и network sections уже задают ожидаемую
  структуру editor payload.
- `01-121` — channel working hours уже входят в channel editor contract и
  должны остаться после split.

## Что сделать

- Разделить transport-слой минимум на `channels CRUD`, `bot credentials`,
  `Telegram diagnostics/test-message`, `channel notifications`.
- Перенести mapping и helper orchestration туда, где это уменьшает связность
  controller-слоя без смены внешнего API.
- Синхронизировать WebMvc coverage под новую controller-разбивку.
- Сохранить текущие API-контракты для фронта и runtime-операций каналов.

## Критерии готовности

- `ChannelApiController` больше не держит все channel-adjacent use-cases в одном
  controller-классе.
- Основные channel transport responsibilities разнесены по более узким
  controllers/services.
- Существующие channel UI/runtime сценарии продолжают работать без смены API.

- После file changes создать отдельный changelog-файл в `ai-context/changelog` и включить в него пользовательский промт, который привёл к изменению.
