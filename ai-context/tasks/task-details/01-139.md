# Задача [01-139]

## Короткое описание

Перенести список «Мои диалоги» в правую часть окна диалога и показывать там новые неназначенные обращения отдельной секцией.

## Цель

Сделать так, чтобы оператор, находясь внутри открытого диалога, не терял новые входящие обращения: список его диалогов должен оставаться видимым справа, а новые неназначенные обращения должны появляться в этом же списке отдельным верхним блоком «Новые».

## Контекст

- Пользователь описал операционную проблему: при открытом окне диалога оператор не видит, что появился новый неназначенный диалог, и может пропустить обращение.
- На странице диалогов уже существует блок «Мои диалоги» с секциями «Неотвеченные» и «В работе», встроенный в шаблон `templates/dialogs/index.html` и обслуживаемый runtime-логикой `dialogs.js` и `dialogs-my-dialogs-runtime.js`.
- Для реализации задачи потребуется одновременно скорректировать layout окна диалога и обновить клиентскую группировку/отрисовку списка, чтобы новые неназначенные обращения показывались в том же видимом списке без выхода из текущего диалога.

## Что сделать

- Перенести блок «Мои диалоги» в правую часть окна открытого диалога, сохранив читаемость основной области переписки и существующих управляющих блоков.
- Добавить в этот же список отдельную секцию «Новые» для новых неназначенных обращений.
- Показывать секцию «Новые» выше секций «Неотвеченные» и «В работе».
- Обеспечить обновление списка во время работы с уже открытым диалогом, чтобы новые неназначенные обращения становились видимыми без ручного возврата в общий список.
- Сохранить существующую логику и визуальное разделение для уже назначенных обращений.

## Критерии готовности

- При открытом окне диалога оператор видит блок «Мои диалоги» справа от основной области переписки.
- Если приходит новый неназначенный диалог, он появляется в этом же списке без необходимости закрывать текущий диалог.
- Новые неназначенные диалоги отображаются отдельной секцией «Новые» и идут выше секций «Неотвеченные» и «В работе».
- Текущие группировки «Неотвеченные» и «В работе» продолжают работать без регрессии.
