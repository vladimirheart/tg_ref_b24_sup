# Задача [01-026]

## Заголовок

Добавить sensitive-topic слой и policy-ограничения для рискованных тем клиентских обращений

## Контекст

Эвристики по отдельным словам не покрывают бизнес-риски. Нужна формальная
классификация чувствительных тем до retrieval, чтобы ограничивать auto-reply на
финансовых, юридических и safety-кейсах.

## Что нужно сделать

- создать rule-based слой классификации sensitive topics в `AiPolicyService`;
- ввести и заполнить таблицу `ai_agent_sensitive_patterns`;
- ввести policy mapping по темам: `assist_only` или `escalate_only`;
- учесть high-risk категории: refunds, money, PII, legal, food safety/allergy,
  delivery dispute, aggressive complaint, VIP;
- добавить trace-поля sensitive routing в event log.

## Критерии готовности

- чувствительные темы выявляются до retrieval и влияют на decision;
- для рискованных тем auto-reply блокируется согласно policy;
- паттерны настраиваются через БД, а не только хардкодом;
- есть тесты на positive/negative сценарии sensitive routing.
