# Задача [01-096]

Починить компиляцию `spring-panel` после SQLite-фикса паспорта объекта: текущая версия `ObjectPassportService` не собирается из-за `unreachable statement` в методах вставки.

## Цель

- вернуть успешную компиляцию `spring-panel`;
- сохранить SQLite-совместимое получение `id` после `INSERT`;
- не менять API-контракт страницы паспорта объекта.

## Контекст

- пользователь перезапустил `run-windows.bat` и получил compile error на `ObjectPassportService.java:[139,9] unreachable statement` и `[162,9] unreachable statement`;
- ошибка возникла после замены `getGeneratedKeys()` на `last_insert_rowid()`, когда в методах остались старые `throw` после гарантированного `return`.

## Что сделать

- убрать недостижимые строки из `ObjectPassportService`;
- убедиться, что `spring-panel\\mvnw.cmd -q -DskipTests compile` снова проходит;
- зафиксировать follow-up отдельным changelog-файлом.

## Критерии готовности

- `run-windows.bat` больше не падает на компиляции `ObjectPassportService`;
- SQLite-логика сохранения остаётся на `last_insert_rowid()`;
- финальный ответ содержит `Резюме для коммита`.
