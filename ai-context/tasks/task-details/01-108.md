# Задача [01-108]

Развести два разных сценария Telegram-сетевой настройки: обычный forward proxy для выхода в `api.telegram.org` и альтернативный Bot API mirror/base URL, который нельзя корректно использовать как HTTP CONNECT proxy.

## Цель

- перестать показывать в панели, что Telegram-бот «запущен», если Spring-процесс жив, но сам Telegram runtime не смог пройти `getMe` или поднять long polling;
- добавить в настройки канала явное поле `Telegram Bot API base URL` для mirror/self-hosted endpoint;
- использовать этот `base URL` и в panel-side запросах (`getMe`, тестовые отправки), и в отдельном Java runtime бота.

## Контекст

- пользователь указал в настройках бота proxy `https://telegram.ftl-dev.ru:443`;
- в логах Telegram runtime получает `Unable to tunnel through proxy ... 405 Not Allowed`, хотя прямой запрос `https://telegram.ftl-dev.ru/bot<TOKEN>/getMe` работает;
- это означает, что endpoint ведёт себя как Bot API mirror/reverse proxy, а не как forward proxy для `CONNECT api.telegram.org:443`;
- текущий runtime проглатывает часть Telegram startup-ошибок и оставляет процесс живым, поэтому панель считает бот запущенным.

## Что сделать

- добавить в `spring-panel` поддержку `platform_config.base_url` для Telegram-канала;
- использовать этот `base_url` в `ChannelApiController` для `getMe` и тестовой отправки;
- прокинуть `TELEGRAM_BOT_API_BASE_URL` в окружение отдельного bot-процесса;
- в `bot-telegram` настроить `DefaultBotOptions.setBaseUrl(...)`;
- перестать скрывать фатальные Telegram startup-ошибки: если `getMe`/long polling не поднялись, runtime должен падать с понятным сообщением, чтобы панель видела `error`, а не `running`;
- зафиксировать изменения тестами и append-only changelog.

## Критерии готовности

- канал Telegram может использовать mirror вида `https://host` через отдельное поле `base URL`, без подмены proxy-настроек;
- panel-side `getMe` и тестовая отправка используют тот же `base URL`;
- отдельный Telegram bot runtime использует `TELEGRAM_BOT_API_BASE_URL`;
- если Telegram runtime не может подключиться, запуск канала возвращает ошибку и не показывает ложный статус «бот запущен»;
- финальный ответ содержит `Резюме для коммита`.
