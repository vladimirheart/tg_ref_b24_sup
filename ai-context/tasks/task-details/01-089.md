# Задача [01-089]

Починить падение `spring-panel` на старте после добавления миграции для `users.photo`: Flyway находит две миграции с версией `20` и не даёт приложению подняться.

## Цель

- восстановить запуск `spring-panel`;
- убрать конфликт версий между SQL- и Java-миграциями Flyway;
- сохранить миграцию `users.photo`, но с уникальной версией.

## Контекст

- в `src/main/resources/db/migration/sqlite` уже есть `V20__password_reset_requests.sql`;
- для аватара пользователя была добавлена Java-миграция `V20__add_photo_column_to_users`;
- при старте Flyway падает с `Found more than one migration with version 20`.

## Что сделать

- перенумеровать Java-миграцию `users.photo` на свободную версию после существующих SQLite-миграций;
- удалить старый конфликтующий файл;
- прогнать `clean` + сборку и быстрый запуск для проверки отсутствия ошибки Flyway.

## Критерии готовности

- Flyway больше не сообщает о двух миграциях с версией `20`;
- `spring-panel` проходит этап миграций и инициализации контекста;
- миграция `users.photo` остаётся в проекте с уникальной версией.

## Дополнительные условия

- после file changes создать отдельный changelog в `ai-context/changelog`;
- в финальном ответе подготовить `Резюме для коммита`.
