# Задача [01-106]

Восстановить загрузку фото в паспорте объекта и отображение титульной фотографии в шапке карточки.

## Цель

- убрать `500` на `POST /api/object_passports/{id}/photos`;
- сделать рабочими photo-операции страницы паспорта: upload, update category/caption, delete и download;
- гарантировать, что титульное фото реально появляется в шапке после успешной загрузки или перевода категории в `title`.

## Контекст

- frontend `passports/new.html` уже давно отправляет запросы:
  - `POST /api/object_passports/{id}/photos`;
  - `PATCH /api/object_passports/photos/{photoId}`;
  - `DELETE /api/object_passports/photos/{photoId}`;
- в `logs/spring-panel.log` зафиксировано: `No static resource api/object_passports/8/photos`, то есть backend route для upload отсутствовал;
- превью в шапке уже реализовано на фронте и ожидает элемент `photos[]` с `category === 'title'` и заполненным `url`;
- текущий backend `ObjectPassportService` хранил только базовый JSON паспорта и не реализовывал отдельный photo-flow.

## Что сделать

- добавить photo-storage для паспортов объектов с сохранением изображений в `attachments`;
- расширить `ObjectPassportApiController` photo-endpoint-ами и download-route;
- расширить `ObjectPassportService` сохранением метаданных фото в `details.photos`, canonical `url` и логикой одного титульного фото;
- зафиксировать изменение отдельным changelog-файлом с пользовательским промтом.

## Критерии готовности

- загрузка фото на странице паспорта больше не падает на отсутствии backend-маршрута;
- после upload/update фронт получает валидный `photos[]` список с `url`, который может отрисоваться в карточках и в шапке;
- титульной одновременно остаётся только одна фотография;
- финальный ответ содержит `Резюме для коммита`.
