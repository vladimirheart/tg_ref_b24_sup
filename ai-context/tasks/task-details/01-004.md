# Задача [01-004]

## Короткое описание

Исправить сетевую доступность Telegram API для процесса бота через явное применение proxy-роута.

## Цель

Сделать так, чтобы `bot-telegram` использовал сетевой маршрут, заданный в настройках интеграции (proxy/vpn), и не зависел от сетевого профиля браузера.

## Контекст

- В логах `logs/support-bot-process.log` фиксируются `Connect timed out` и `NoHttpResponseException` при обращении к `api.telegram.org:443`.
- Процесс бота стартует из панели через `BotProcessService` и получает сетевые env-переменные из `IntegrationNetworkService`.
- `SupportBot` использовал конструктор `TelegramLongPollingBot(token)` без явной proxy-конфигурации.

## Что сделать

- Подключить в `SupportBot` явную конфигурацию `DefaultBotOptions` на основе `APP_NETWORK_PROXY_*`.
- Смаппить схемы `socks4/socks5/vless/http/https` в `DefaultBotOptions.ProxyType`.
- Оставить `NO_PROXY`, если прокси не задан.
- Добавить лог о примененном proxy-маршруте.

## Критерии готовности

- `bot-telegram` компилируется.
- При заданном proxy-route бот поднимается с proxy-настройками в `DefaultBotOptions`.
- В коде есть читаемая и безопасная деградация на `NO_PROXY`.
