# Задача [01-088]

Починить сохранение аватара пользователя в настройках: фото успешно загружается в upload endpoint, но при сохранении карточки API отвечает `Нет данных для обновления`, и ссылка на аватар не попадает в `users`.

## Цель

- обеспечить сохранение загруженного аватара в карточке пользователя;
- убрать зависимость API от stale-кэша колонок `users`;
- гарантировать наличие колонки `users.photo` в SQLite-схеме.

## Контекст

- фронтенд загружает изображение и получает URL от `/api/users/photo-upload`;
- при сохранении формы backend `updateUser(...)` собирает список `updates` только по колонкам из `userColumns`;
- `userColumns` кэшируется в конструкторе контроллера, а для старых БД колонка `photo` может отсутствовать вовсе.

## Что сделать

- перевести операции `AuthManagementApiController` на актуальный `loadUserColumns()` вместо одноразового снимка схемы;
- добавить SQLite-миграцию, которая создаёт `users.photo`, если колонка ещё не существует;
- проверить компиляцию `spring-panel`.

## Критерии готовности

- после загрузки фото и сохранения карточки поле `photo` попадает в обновление пользователя;
- API больше не отвечает `Нет данных для обновления` в сценарии изменения только аватара;
- при рестарте проекта Flyway добавляет колонку `users.photo` в старые SQLite-базы.

## Дополнительные условия

- после file changes создать отдельный changelog в `ai-context/changelog`;
- в финальном ответе подготовить `Резюме для коммита`.
