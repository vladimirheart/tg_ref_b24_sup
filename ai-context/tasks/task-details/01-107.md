# Задача [01-107]

Восстановить страницу каталога паспортов объектов `/object-passports`, потому что она показывает пустой список даже при существующих карточках `/object-passports/{id}`.

## Цель

- вернуть отображение реально сохранённых паспортов объектов в каталоге;
- убрать ложную привязку листинга к каталогу ИТ-оборудования;
- сохранить существующий переход из списка в карточку по `id`.

## Контекст

- пользователь видит, что `/object-passports/9` и `/object-passports/1` открываются, но `/object-passports` пуст;
- `ManagementController.passports()` загружал `equipmentRepository.findAll()`, то есть вообще не таблицу `object_passports`;
- шаблон `passports/list.html` тоже был собран как таблица оборудования (`Тип / Производитель / Модель / Серийный номер`), а не как каталог паспортов.

## Что сделать

- добавить в `ObjectPassportService` метод получения списка паспортов из `object_passports` + `objects`;
- переключить `ManagementController` на этот источник;
- обновить `passports/list.html` под реальные поля паспорта;
- зафиксировать изменение отдельным changelog-файлом с пользовательским промтом.

## Критерии готовности

- `/object-passports` показывает существующие карточки паспортов;
- каждая строка открывает корректный `/object-passports/{id}`;
- `spring-panel` продолжает компилироваться после правки;
- финальный ответ содержит `Резюме для коммита`.
