зафиксировать некорректные тезисы и обновить документы.

факты, скорректировать неточные метрики и переформулировать спорные задачи.

- В документах уже выявлены расхождения с кодом (`@EnableScheduling`,

- По `TaskApiController` есть прямое использование entity в API.
- Централизованная обработка ошибок уже есть, но покрытие ограниченное.

## Что сделать

- Проверить утверждения документов по текущему состоянию кода.
- Обновить неточные формулировки и метрики в audit/plan-документах.
- Добавить отдельный документ валидации с таблицей: тезис, факт, решение.

## Критерии готовности

- Обновленные документы не содержат фактически неверных утверждений.
- Для ключевых правок есть проверяемые ссылки на файлы/модули.
- Новые задачи в backlog соответствуют реальному техническому долгу.
