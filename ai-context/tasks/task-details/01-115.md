# Задача [01-115]

## Короткое описание

Доработать шаблоны вопросов в настройках: добавить обязательность для свободных
вопросов и починить реальное сохранение шаблонов.

## Цель

Сделать question templates в settings рабочими end-to-end: свободный вопрос
должен поддерживать обязательность ответа, а сохранённый шаблон должен реально
оставаться в системе после перезагрузки страницы.

## Контекст

- Пользователь указал на две проблемы:
  свободный вопрос требует явной настройки обязательности и необязательности,
  а новые шаблоны визуально сохраняются, но исчезают после повторного открытия
  страницы настроек.
- По changelog `2026-05-27_114538_bot-question-templates-required-and-persistence.md`
  задача включает не только UI, но и фактическое сохранение в `settings`,
  а также поддержку поля `required` в bot-core и runtime-ботах.
- Эта задача теперь считается baseline для `01-128` и `01-129`: dialog
  template editors и их runtime hooks нельзя сломать при дальнейшем split.

## Что сделать

- Добавить в editor question templates явную настройку обязательности для
  свободного вопроса.
- Протянуть флаг `required` через нормализацию, сериализацию и runtime DTO.
- Исправить сохранение шаблона так, чтобы он реально записывался в `settings`
  и возвращался после перезагрузки страницы.
- Поддержать runtime-сценарий пропуска необязательного вопроса в ботах.

## Критерии готовности

- Для свободного вопроса можно указать, обязателен он или нет.
- Сохранённый question template остаётся в настройках после перезагрузки.
- Bot runtime учитывает обязательность вопроса и умеет пропускать
  необязательный свободный вопрос.

## Обязательные дополнительные условия

- В финальном ответе подготовить `Резюме для коммита` по формату из `ai-context/baseline/guides/tasks.md`.
- После file changes создать отдельный changelog-файл в `ai-context/changelog` и включить в него пользовательский промт, который привёл к изменению.
