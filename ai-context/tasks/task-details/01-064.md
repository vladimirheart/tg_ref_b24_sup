# Задача [01-064]

Уточнить chain-лицензию RMS и добавить экспорт Excel с копированием адреса.

## Цель

Довести общий список RMS до рабочего состояния для `chain` и `kiosk`, убрать устаревшие сообщения о fastfood-лицензии у chain-записей и добавить удобные операционные действия: экспорт в Excel с выбором колонок и копирование адреса RMS.

## Контекст

- После `01-060` и `01-063` список RMS уже различает типы серверов, но для chain нужно использовать не `FullEdition (Server)`, а `Chain (Store Connector (1 RMS))` с `id=1300`.
- В части строк ещё может отображаться старое сообщение про `RMS (Front Fast Food) id=100`, сохранённое от предыдущих запусков.
- Для `IIKO_RMS` kiosk-лицензию нужно показывать только если она реально присутствует.
- Пользователю нужен экспорт текущего списка RMS в Excel с выбором столбцов и быстрым копированием адреса RMS.

## Что сделать

- Переключить chain-таргет лицензии на `Chain (Store Connector (1 RMS))` с `id=1300`.
- Убрать из общего списка устаревшее fastfood-сообщение для chain-записей и показывать нейтральный текст до следующего корректного refresh.
- Показывать kiosk-бейдж в колонке даты окончания лицензии только когда количество этой лицензии больше нуля.
- Добавить модалку экспорта RMS в Excel с выбором колонок и выгрузкой текущего видимого списка.
- Добавить действие копирования адреса RMS по строке списка.

## Критерии готовности

- Chain-записи в списке работают по лицензии `Chain (Store Connector (1 RMS)) [id=1300]`.
- В колонке статуса лицензии у chain больше не отображается старое сообщение про `RMS (Front Fast Food) id=100`.
- Kiosk-лицензия показывается только когда она есть.
- На странице RMS есть экспорт в Excel с выбором колонок.
- У каждой строки RMS есть действие копирования адреса.
