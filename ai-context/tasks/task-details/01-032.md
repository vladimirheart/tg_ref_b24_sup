# Задача [01-032]

## Заголовок

Подключить LLM как контролируемый слой (parser/rewrite/composer/explainer) с output guard

## Контекст

LLM должен быть добавлен только после policy/retrieval foundation и использоваться
как ограниченный исполнитель в рамках доверенных evidence, а не как автономный
решатель бизнес-правил.

## Что нужно сделать

- внедрить 4 роли LLM:
  intent parser, query rewrite, response composer, operator explainer;
- запретить LLM принимать policy-решения и отправлять ответы без trusted evidence;
- реализовать output guard:
  запрет на выдуманные политики, небезопасные обещания по refund/деньгам,
  раскрытие PII и ссылки на несуществующие источники;
- подключить rollout-контур с feature flags и A/B между assist-only и
  selective auto-reply;
- подготовить fallback на deterministic response при ошибке LLM слоя.

## Критерии готовности

- LLM не нарушает policy boundaries и не обходит decision engine;
- каждый сгенерированный ответ привязан к подтвержденному evidence;
- output guard перехватывает unsafe/hallucinated формулировки;
- rollout управляется конфигом и поддерживает безопасный откат.
