# Задача [01-094]

Восстановить backend API для страницы паспорта объекта, потому что frontend уже отправляет `POST/PUT/GET` на `/api/object_passports/**`, а в текущем состоянии репозитория соответствующего контроллера нет.

## Цель

- убрать `500` при сохранении паспорта, вызванный отсутствием маршрута `/api/object_passports`;
- вернуть минимальный рабочий цикл `создать -> открыть -> снова сохранить`;
- не переписывать весь функционал паспортов, а поднять совместимый минимальный API для базовых сценариев.

## Контекст

- в `logs/spring-panel.log` зафиксировано `No static resource api/object_passports`, то есть запрос попадает в static resource handler, а не в контроллер;
- frontend `passports/new.html` уже давно ожидает полный namespace `/api/object_passports/**`;
- в `objects.db` уже есть таблицы `objects` и `object_passports`, где у `object_passports` есть поле `details`, подходящее для хранения JSON payload страницы;
- страница существующего паспорта рендерилась только с `{ "is_new": false }`, поэтому после redirect с create она открывалась без догрузки сохранённых данных.

## Что сделать

- добавить `ObjectPassportApiController` с маршрутами:
  - `POST /api/object_passports`;
  - `PUT /api/object_passports/{id}`;
  - `GET /api/object_passports/{id}`;
  - минимальные `GET /cases` и `GET /tasks` с пустыми payload;
- добавить `ObjectPassportService`, который сохраняет базовый payload в `objects.db` через JSON `details`;
- создавать/обновлять связанную запись в `objects`, чтобы соблюдать `object_id` foreign key;
- подправить `ensurePassportDataLoaded()` на фронте, чтобы existing-паспорт догружался с API, если сервер не передал содержательный payload;
- добавить WebMvc smoke test на новый API namespace.

## Критерии готовности

- сохранение нового паспорта больше не падает на отсутствующем маршруте;
- после создания страница `/object-passports/{id}` может загрузить сохранённые данные через `GET /api/object_passports/{id}`;
- основная компиляция `spring-panel` проходит;
- если глобальный test-compile не блокируется сторонними ошибками репозитория, новый WebMvc test проходит.
