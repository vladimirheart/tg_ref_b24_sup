# Задача [01-128]

## Короткое описание

Вынести page-shell и inline bootstrap страницы настроек в отдельный settings-page-shell runtime.

## Цель

Убрать из `settings/index.html` роль giant page-bootstrap entrypoint, чтобы
template отвечал прежде всего за markup, а инициализация страницы жила в
отдельном client-side модуле с явными точками входа.

## Контекст

- По audit-срезу `settings/index.html` сейчас держит около `24659` строк.
- Внутри файла остаются inline `<style>`, большой inline `<script>` и как
  минимум `37` inline `onclick`.
- Основной coupling на первом шаге живёт не только в markup, но и в page-shell
  bootstrap: modal stacking, tile activation, theme form sync, DOMContentLoaded
  orchestration и глобальные entry-points живут прямо в template.
- Эта задача является первым пакетом `Track A` из `01-024` и должна идти до
  более глубокого split внутренних settings subdomains.

## Что сделать

- Выделить из `settings/index.html` page-shell runtime в отдельный JS-модуль
  наподобие `settings-page-shell.js`.
- Перенести туда инициализацию tile-behavior, primary modal shell, body lock,
  z-index management и соседний bootstrap страницы.
- Убрать из template inline bootstrapping там, где это можно сделать через
  `data-*` hooks и `DOMContentLoaded` внутри внешнего модуля.
- Сохранить текущее пользовательское поведение modals/sheets/tile-open flow без
  визуального редизайна.

## Критерии готовности

- `settings/index.html` больше не держит основной page-shell bootstrap внутри
  giant inline script-блока.
- Modal stacking, body lock и tile activation продолжают работать без
  regression.
- Появляется отдельный внешний runtime entrypoint для settings page shell.

## Обязательные дополнительные условия

- В финальном ответе подготовить `Резюме для коммита` по формату из `ai-context/baseline/guides/tasks.md`.
- После file changes создать отдельный changelog-файл в `ai-context/changelog` и включить в него пользовательский промт, который привёл к изменению.
