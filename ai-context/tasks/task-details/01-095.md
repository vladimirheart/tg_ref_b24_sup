# Задача [01-095]

Починить создание паспорта объекта в `ObjectPassportService`: текущее сохранение падает на SQLite JDBC с `SQLFeatureNotSupportedException`, потому что драйвер не поддерживает `PreparedStatement.getGeneratedKeys()`.

## Цель

- убрать `500` на `POST /api/object_passports` при создании нового паспорта;
- сделать получение `id` новой записи совместимым с реальным SQLite-драйвером проекта;
- не менять контракт frontend API, а исправить только backend-слой сохранения.

## Контекст

- в `logs/spring-panel.log` зафиксирован `Caused by: java.sql.SQLFeatureNotSupportedException: not implemented by SQLite JDBC driver`;
- падение происходит в `ObjectPassportService.insertObject()` и дальше блокирует весь сценарий создания паспорта;
- в репозитории уже есть примеры SQLite-совместимого получения id через `SELECT last_insert_rowid()`.

## Что сделать

- убрать использование `Statement.RETURN_GENERATED_KEYS` и `getGeneratedKeys()` из `ObjectPassportService`;
- после `INSERT` получать id через `SELECT last_insert_rowid()` на том же `Connection`;
- убедиться, что сборка `spring-panel` проходит после изменения.

## Критерии готовности

- `POST /api/object_passports` больше не падает из-за `getGeneratedKeys()` на SQLite;
- `spring-panel` компилируется без ошибок;
- изменение зафиксировано отдельным changelog-файлом.

## Дополнительные условия

- не трогать пользовательские данные и несвязанные dirty-файлы;
- в финальном ответе подготовить `Резюме для коммита`.
