# Задача [01-011]

## Короткое описание

Добавить failover-цепочки для маршрутов proxy/vpn и перевести MAX-бот на long polling.

## Цель

- Разрешить использовать несколько сетевых профилей в одном маршруте (по порядку переключения).
- Добавить проверку доступности маршрутов и TTL недоступности.
- Убрать зависимость MAX-бота от webhook в активном режиме и запустить получение событий через long polling.
- Сохранить webhook-код в проекте в закомментированном виде для быстрого возврата.

## Что сделано

- `IntegrationNetworkService` расширен: поддержка `profile_ids`, `failover_downtime_seconds`, health-check и переключение на следующий профиль.
- В UI настроек сети добавлены цепочки профилей (multi-select) и настройка TTL недоступности для проекта/ботов/канала.
- `bot-max` переведён на long polling:
  - добавлен `MaxLongPollingLifecycle`;
  - `MaxApiClient` получил метод `fetchUpdates`;
  - webhook-аннотации в `MaxWebhookController` закомментированы (код не удалён).
- В UI подсказках для MAX отражён режим long polling.
