# Задача [01-072]

## Короткое описание

Вынести `iikoServer`-источники локаций в страницу настроек и не смешивать их с `iiko biz`.

## Цель

Развести в UI и в storage два разных контура:

- `iiko biz / transport` для интеграций transport API;
- `iikoServer` для live-структуры локаций и синхронизации департаментов.

## Контекст

- После `01-071` location runtime должен иметь чёткий source-of-truth, но этого недостаточно,
  если пользователь по-прежнему видит смешанный UX и не понимает, какие креды к какому сценарию относятся.
- На странице настроек уже есть вкладка `Синхронизация iikoServer`, где должны жить отдельные
  источники для location sync.
- Внутри `settings/index.html` и соседнего settings runtime это один из важных baseline-срезов
  для `Track A3: locations tree / sync / wizard`.
- Задача не про новый внешний дизайн, а про корректную декомпозицию настроек источников и
  уменьшение операционной двусмысленности.

## Пересечения с backlog

- `01-071` задаёт source boundary; `01-072` делает эту границу явной в settings UI и persistence.
- `01-075` / `01-076` зависят от того, что `iikoServer`-источники уже живут отдельно и имеют
  собственную credential model.
- `01-128` / `01-129` не должны заново смешивать location-source editor с transport/editor flow
  во время будущего runtime split.

## Что сделать

- Выделить на странице настроек отдельный editor для `iikoServer`-источников локаций.
- Хранить и редактировать `iikoServer` source settings отдельно от `iiko biz` / transport profile settings.
- Уточнить тексты, поля и действия так, чтобы оператор явно понимал, что эти источники
  участвуют в синхронизации департаментов, а не в transport API вызовах.
- Сохранить совместимость с existing location sync/status flow.

## Критерии готовности

- Источники `iikoServer` редактируются на странице настроек как отдельный bounded-блок.
- В UI и в backend больше не смешиваются креды/поля `iikoServer` и `iiko biz`.
- Пользователь может понять по интерфейсу, какой профиль отвечает за live-локации, а какой —
  за transport API.
