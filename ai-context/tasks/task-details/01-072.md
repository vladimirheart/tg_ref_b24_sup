# Задача [01-072]

Вынести настройку `iikoServer`-источников для структуры локаций на страницу настроек и убрать зависимость этого функционала от user-scoped `iiko biz`-кредитов в `AI Ops`.

## Цель

Разделить два контура:

- `iiko biz` / автоматизация скидок остаётся в `AI Ops`;
- `iikoServer`-источники департаментов для структуры локаций настраиваются в `Настройки -> Структура локаций`.

## Контекст

- Предыдущая правка перевела live-локации на `iikoServer API`, но временно использовала активный `iiko URL`-профиль из automation-раздела.
- Пользователь уточнил, что это некорректное место: там лежат креды для `iiko biz`, а не для `iikoServer`.
- Нужно проверить, что логика `iiko biz` не сломана после отвязки.

## Что сделать

- Добавить в страницу `Настройки` отдельный UI-блок для источников `iikoServer` внутри структуры локаций.
- Хранить эти источники в shared settings отдельно от automation credentials.
- Перевести `IikoDepartmentLocationCatalogService` на новый shared settings source.
- Убрать зависимость live-локаций от `EmployeeDiscountAutomationCredentialService`.
- Проверить, что контур `AI Ops / employee-discounts / iiko biz` компилируется и не ломается от этой развязки.

## Критерии готовности

- Адреса/логины/секреты `iikoServer` для локаций настраиваются со страницы настроек.
- `AI Ops` больше не используется как источник `iikoServer` для структуры локаций.
- Live-синхронизация локаций работает через новый shared settings source.
- `iiko biz`-контур продолжает собираться и не зависит от новой настройки.
