# Задача [01-009]

## Короткое описание

Добавить реальную проверку готовности процесса бота после старта.

## Цель

После запуска бота панель должна возвращать успешный статус только если процесс
действительно прошел стартовое окно и подтвердил готовность, а не просто был
создан на уровне `ProcessBuilder`.

## Контекст и ограничения

- Затронута область запуска бот-процессов в `spring-panel`.
- Нужно сохранить совместимость с текущим управлением ботами через Maven
  модули `java-bot`.
- В рабочем репозитории отсутствуют project-specific правила в
  `ai-context/rules`, поэтому применяются baseline-правила.

## Применимые правила

- `ai-context/baseline/ai-rules/002_context-before-analysis.md`
- `ai-context/baseline/ai-rules/003_task-execution-protocol.md`
- `ai-context/baseline/ai-rules/005_append-only-changelog.md`
- `ai-context/baseline/ai-rules/006_project-architecture-rules.md`

## Связанные файлы

- `spring-panel/src/main/java/com/example/panel/service/BotProcessService.java`
- `spring-panel/src/test/java/com/example/panel/service/BotProcessServiceTest.java`

## Что сделано

- Добавлено ожидание readiness после `builder.start()`.
- Успех фиксируется только после появления Spring Boot маркера
  `Started ...Application`.
- Ошибки старта извлекаются из `APPLICATION FAILED TO START` и возвращаются в
  понятном сообщении.
- Добавлены тесты для успешного старта, падения на старте и таймаута readiness.
