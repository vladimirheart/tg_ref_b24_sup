# Задача [01-002]

Этот файл фиксирует детализацию задачи из `tasks/task-list.md`.

## Короткое описание

Усилить централизованную обработку ошибок для REST API
(`RestControllerAdvice` + единый формат ответов).

## Цель

Сделать поведение API ошибок предсказуемым и единообразным для клиентов фронта
и интеграций.

## Контекст

- Есть `RestExceptionHandler`, но он покрывает только часть исключений.
- В контроллерах встречаются разные форматы ошибок (`ok=false`,
  `success=false`, `ResponseStatusException` без единого тела).
- Важно не ломать текущие рабочие сценарии.

## Что сделать

- Перевести обработчик на `@RestControllerAdvice`.
- Ввести единый DTO/формат ошибки для централизованных обработчиков.
- Добавить обработку `ResponseStatusException` и fallback `Exception`.
- Обновить `TaskApiController`, чтобы 404/ошибки проходили через advice.

## Критерии готовности

- Исключения из API возвращают единый JSON-формат.
- `ResponseStatusException` отдает корректный HTTP статус и текст ошибки.
- Сборка `spring-panel` проходит успешно.

## Обязательные дополнительные условия

- После изменений создать отдельный файл в `ai-context/changelog/`.
- В финальном ответе подготовить секцию `Резюме для коммита`.
