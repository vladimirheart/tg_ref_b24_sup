# Задача [01-146]

## Короткое описание

Починить UX и фактическое прикрепление медиа в ответе оператора.

## Цель

Убрать ложные popup-уведомления о якобы добавленном медиа, сделать состояние pending-вложений явным прямо в composer и исключить ситуацию, когда оператор не понимает, прикрепился ли файл к сообщению.

## Контекст

- В dialogs composer есть отдельный flow выбора/вставки медиа до фактической отправки.
- Сейчас при выборе файла и вставке скриншота фронт показывает success/info popup, но не даёт оператору явного inline-признака, что вложение действительно ожидает отправки.
- После фактической отправки media flow также показывает success toast, хотя пользователь просит убрать такие сообщения и оставить только реальные ошибки.

## Что сделать

- Убрать success popup при выборе файла и при успешной отправке media.
- Добавить явную inline-индикацию pending-вложений в details/workspace composer.
- Проверить, что после очистки/отправки индикатор возвращается в исходное состояние.
- Зафиксировать изменение в append-only changelog.

## Критерии готовности

- Выбор файла не показывает ложный popup об успешном добавлении.
- Оператор видит по composer, что вложения действительно ожидают отправки.
- После успешной отправки success popup не показывается.
- Ошибки отправки медиа продолжают показываться.

## Обязательные дополнительные условия

- В финальном ответе подготовить `Резюме для коммита`.
- После file changes создать отдельный changelog-файл в `ai-context/changelog`.
