# Задача [01-134]

## Короткое описание

Выпилить оставшийся backend/test слой `PublicForm` после удаления публичных
форм из проекта.

## Цель

- убрать из живого кода и тестового контура остаточный `PublicForm`-subsystem,
  который больше не соответствует текущему product scope;
- зачистить связанные watcher/audit-ветки и integration/template tests,
  завязанные на `public_form_*`;
- зафиксировать removal как отдельный пакет, а не смешивать его с cleanup
  settings runtime из `01-129`.

## Контекст

- в рамках `01-129` из `settings/index.html` уже удалены JS-хвосты
  `public_form`; это был только локальный cleanup settings-страницы, а не
  удаление всего subsystem;
- пользователь отдельно подтвердил, что публичные формы выпилены из проекта и
  не интересуют текущий продуктовый этап;
- по повторной сверке на `2026-06-26` прямых текстовых references
  `PublicForm|public_form|publicForm|PublicShell` внутри
  `spring-panel` и `java-bot` больше не найдено; исходный inventory для этой
  задачи оказался историческим и уже не совпадает с текущим source-state;
- отдельные исторические задачи и документы уже касались этого контура:
  `01-086`, `01-129`,
  `docs/ARCH_UI_REFACTORING_ROADMAP_2026-04-15.md`,
  `docs/ARCHITECTURE_AUDIT_2026-04-08.md`.

## Текущий статус

- Статус задачи: `🟣 ожидает ручной проверки`.
- По состоянию на `2026-06-26` практический removal-пакет для живого
  backend/test source уже, по сути, отыгран вне этой карточки: дополнительная
  сверка не показала прямых `PublicForm/public_form/PublicShell` совпадений в
  `spring-panel` и `java-bot`.
- На текущем этапе задача больше не про удаление кода, а про корректную
  фиксацию результата: не держать в backlog устаревший inventory и не
  планировать removal того, чего в source уже нет.

## Что сделать

- зафиксировать в task detail, что прямой `PublicForm` source-scope уже
  отсутствует и исходный inventory задачи устарел;
- при необходимости точечно обновлять только backlog/операционный контекст,
  если в нём остались ссылки на уже удалённый активный код;
- не переписывать исторические audit/changelog-документы массово, если это не
  требуется отдельной задачей.

## Критерии готовности

- `rg` по `PublicForm|public_form|publicForm|PublicShell` не показывает
  прямых source references в `spring-panel` и `java-bot`;
- task detail не содержит устаревших утверждений о файлах, которых уже нет в
  актуальном source-state;
- исторические mentions в `docs/` и `ai-context/` трактуются именно как
  исторический след, а не как активный runtime scope.

## Обязательные дополнительные условия

- В финальном ответе подготовить `Резюме для коммита` по формату из
