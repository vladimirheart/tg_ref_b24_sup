# Задача [01-052]

## Короткое описание

Починить создание iiko API monitor и расширить диагностику RMS в UI.

## Цель

Устранить runtime-ошибку при создании записи мониторинга iiko API, перевести
RMS-мониторинг на целевую лицензию `RMS (Front Fast Food)` `id=100` и дать
оператору доступ к диагностическим данным из интерфейса.

## Контекст

По `spring-panel.log` причина падения при создании iiko monitor уже подтверждена:
`SQLFeatureNotSupportedException: not implemented by SQLite JDBC driver` в
`IikoApiMonitorRepository.insert()` при использовании `getGeneratedKeys`.
В RMS сейчас срок берётся по самой ранней найденной лицензии без фильтра по
целевой лицензии, а успешный ping не показывает резолвленный IP и не даёт
оператору посмотреть raw-диагностику из UI.

## Что сделать

- Убрать зависимость от `getGeneratedKeys` в SQLite-репозиториях мониторинга.
- Перевести RMS license parsing на целевую лицензию `RMS (Front Fast Food)` с
  `id=100`.
- Сохранять и показывать диагностические данные RMS из UI: raw ping,
  трассировку и отладочный фрагмент ответа по лицензии.
- Показать IP-адрес в статусе успешного ping.

## Критерии готовности

- Создание записи iiko API monitor больше не падает на SQLite insert.
- На RMS-странице срок лицензии считается по `RMS (Front Fast Food)` `id=100`.
- В UI есть отдельный просмотр диагностики RMS.
- `spring-panel\\mvnw.cmd -q -DskipTests compile` проходит успешно.

## Обязательные дополнительные условия

- В финальном ответе подготовить `Резюме для коммита` по формату из
  `ai-context/baseline/guides/tasks.md`.
- После любых file changes создать отдельный changelog-файл в
  `ai-context/changelog` по правилу «одно изменение = один файл».
