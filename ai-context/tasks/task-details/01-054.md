# Задача [01-054]

## Короткое описание

Разобрать свежие ошибки из `spring-panel.log` и починить битое имя RMS после
сохранения пароля на странице контроля.

## Цель

Убрать ситуацию, когда после обновления записи RMS колонка `Имя` затирается
битым значением из ответа `get_server_info.jsp`, и зафиксировать по логу
актуальные runtime-проблемы.

## Контекст

- Лог панели лежит в `C:\Users\SinicinVV\git_h\tg_ref_b24_sup\logs\spring-panel.log`.
- В `refreshLicenseState()` имя RMS перезаписывается результатом
  `fetchServerMetadata()`.
- По свежему логу есть серия `SQLITE_BUSY` на `SPRING_SESSION`, а по UI видно
  mojibake в столбце `Имя`.

## Что сделать

- Проанализировать свежие ошибки в `spring-panel.log` и сформулировать выводы.
- Усилить декодирование `serverName` из `get_server_info.jsp`.
- Не перетирать корректное имя RMS битым значением, если ответ сервера выглядит
  как mojibake.

## Критерии готовности

- После сохранения пароля `Имя` не затирается битой строкой.
- Сборка `mvnw -q -DskipTests compile` проходит.
- В ответе пользователю отражены ключевые выводы по лог-файлу.

## Обязательные дополнительные условия

- В финальном ответе подготовить `Резюме для коммита` по формату из
  `ai-context/baseline/guides/tasks.md`.
