# Задача [01-127]

## Короткое описание

Добавить на dashboard drag&drop и pin для аналитических панелей.

## Цель

Дать пользователю те же возможности управления панелями, что и в референсе: перетаскивание, закрепление сверху и сохранение собственного layout.

## Контекст

- Пользователь приложил референс с drag-and-drop grid и pinning.
- На текущем dashboard есть отдельный аналитический grid из панелей, который логично сделать настраиваемым без перемещения верхнего control-блока с фильтрами и KPI.
- Для задачи достаточно клиентского сохранения layout; серверный профиль пользователя пока не требуется.

## Что сделать

- Добавить аналитическим панелям dashboard уникальные `panelId`.
- Реализовать на фронте `drag&drop` с placeholder и drag-handle.
- Добавить `pin` для закрепления панели в верхней части grid.
- Сохранять и восстанавливать layout из localStorage.

## Критерии готовности

- Панели аналитического grid можно перетаскивать мышью за drag-handle.
- Панели можно закреплять и откреплять, а закреплённые поднимаются вверх.
- Порядок и флаг pin сохраняются между перезагрузками страницы.

## Обязательные дополнительные условия

- В финальном ответе подготовить `Резюме для коммита` по формату из `ai-context/baseline/guides/tasks.md`.
- После file changes создать отдельный changelog-файл в `ai-context/changelog` и включить в него пользовательский промт, который привёл к изменению.
