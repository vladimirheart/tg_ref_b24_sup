# Задача [01-022]

## Заголовок

Ввести общий UI config/preset слой для dashboard, settings и dialogs

## Контекст

В проекте уже накопились сильные визуальные паттерны для `dashboard`,
`settings` и `dialogs`, но они пока живут как набор page-specific решений.
Нужен общий слой presets и shared UI-структур, чтобы страницы опирались на
единые правила, а не на разрозненные стили.

## Что нужно сделать

- добавить общий `ui-config` слой для страниц;
- ввести presets для `dashboard`, `settings` и `dialogs`;
- добавить shared shell/panel паттерны в CSS;
- подключить новый слой к целевым страницам;
- не ломать существующую бизнес-логику страниц.

## Ожидаемый результат

- в проекте появляется техническая основа для page presets;
- страницы получают `data-ui-page` и общие UI-атрибуты;
- базовые layout/panel решения переиспользуются;
- последующие UI-изменения проще делать через общие presets.
