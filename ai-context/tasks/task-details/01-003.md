# Задача [01-003]

Этот файл фиксирует детализацию задачи из `tasks/task-list.md`.

## Короткое описание

Добавить минимальное API versioning для task API через `/api/v1/tasks`
с сохранением обратной совместимости.

## Цель

Подготовить API задач к управляемому развитию контрактов без поломки текущих
клиентов, использующих `/api/tasks`.

## Контекст

- Сейчас `TaskApiController` обслуживает только `/api/tasks`.
- В audit/plan версионирование выделено как отдельный технический долг.
- Нужен безопасный первый шаг без массового рефакторинга остальных API.

## Что сделать

- Добавить маршрут `/api/v1/tasks` на существующий контроллер задач.
- Сохранить поддержку legacy маршрута `/api/tasks`.
- Проверить, что эндпоинты CRUD и комментариев доступны по обоим префиксам.

## Критерии готовности

- Базовые методы task API доступны по `/api/tasks` и `/api/v1/tasks`.
- Существующие вызовы не ломаются.
- Сборка `spring-panel` проходит успешно.

## Обязательные дополнительные условия
