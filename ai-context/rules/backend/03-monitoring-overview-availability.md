# Правило: monitoring overview должен показывать агрегированную доступность

## Статус

Действует.

## Правило

Если в `spring-panel` существует monitoring-страница со списком проверяемых узлов, а сами проверки включают availability/network health state, такая страница должна показывать не только построчный статус, но и общий агрегированный overview доступности.

Минимальный состав overview:

- доля доступных узлов в процентах;
- абсолютные счётчики `up`, `down`, `unknown`, `disabled`;
- визуальный график или stacked-summary, который позволяет быстро увидеть перекос без чтения всей таблицы.

## Область применения

- RMS monitoring overview;
- будущие overview-страницы для SSL / HTTP / DNS / SMTP / iiko availability, если в них появится именно сетевой статус узлов;
- любые новые monitoring dashboards, где у пользователя есть список проверяемых endpoints и нужно быстро оценить общее состояние контура.

## Практический смысл

- оператор сразу видит здоровье контура в целом, а не только статус отдельных строк;
- проще заметить деградацию или массовый инцидент без ручного просмотра таблицы;
- monitoring pages получают единый UX-паттерн: detail diagnostics показывают timeline, а overview pages показывают агрегированную доступность.
