# Правило: Публичные webhook платформенных ботов идут через `spring-panel`

## Статус

Действует.

## Правило

Если бот запускается отдельным локальным процессом, его публичный webhook не
должен публиковаться напрямую из этого процесса наружу. Публичная точка входа
должна жить в `spring-panel`, а panel уже проксирует запрос во внутренний
бот-процесс.

## Для webhook-платформ

- публичный URL строится на origin панели;
- panel принимает внешний webhook без пользовательской аутентификации;
- внутренний бот-процесс слушает только локальный адрес `127.0.0.1`;
- для нескольких бот-процессов используются отдельные локальные порты;
- если webhook привязан к конкретному каналу, идентификатор канала должен быть
  частью публичного URL.

## Практический смысл

- panel остаётся единой публичной точкой входа;
- снижается риск портовых конфликтов между несколькими ботами одной платформы;
- UI может показывать корректный webhook URL, который реально обслуживается;
- внутренние бот-процессы не требуется открывать наружу напрямую.
