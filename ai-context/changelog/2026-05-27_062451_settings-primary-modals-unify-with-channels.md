# Изменения

- В `spring-panel/src/main/resources/templates/settings/index.html` введён общий shell `settings-primary-modal` для верхнеуровневых модалок страницы настроек.
- Основные окна, открывающиеся из плиток настроек, переведены на единый полноформатный паттерн открытия по образцу `Каналы (боты)` вместо смешения обычных centered modal и боковых sheet-модалок.
- Для модалок `Пользователи`, `Настройка диалогов`, `Привязка управляющих`, `Конструктор автоотчётов`, `Настройки оформления`, `Локации`, `Параметры партнёров`, `Юридические лица`, `Подключения IT-блока` и `Настройка вводимых данных` унифицированы классы, размеры dialog и поведение прокрутки.
- JS-инициализация верхнеуровневых модалок вынесена в общий `initSettingsPrimaryModals()`: подсветка активной плитки, body-lock и общий modal lifecycle теперь работают одинаково для всех основных окон страницы.
- В `ai-context/tasks/task-list.md` добавлена задача `01-113` про унификацию основных модалок настроек.

# Проверки

- Автотесты и браузерная сборка не запускались: правка ограничена шаблоном страницы настроек и её runtime-логикой на клиенте.
- Локально проверен дифф разметки и JS-инициализации: все главные `data-bs-target`-модалки страницы теперь помечены `data-settings-primary-modal`.
