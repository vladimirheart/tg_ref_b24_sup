# Notion Change Monitoring And Background Sync

## Что сделано

- Добавлен массовый ручной сценарий `Обновить изменённые` для статей Notion из блока интеграции базы знаний.
- В preview импорта появился статус синхронизации каждой записи: `Новая`, `Изменена в Notion`, `Актуальна`.
- Добавлена кнопка `Выбрать изменённые` в preview-таблице для быстрой загрузки только новых и обновлённых записей.
- На странице базы знаний появился блок мониторинга последней синхронизации Notion с результатами последнего запуска.
- Добавлен фоновый scheduler для периодической синхронизации уже связанных с Notion статей.
- Точечная синхронизация одной статьи из карточки теперь тоже обновляет общий sync-status.
- `knowledge/list.html` переписан в чистом UTF-8 при обновлении UI.

## Почему

- Пользователь не видел, какие статьи реально изменились в Notion и какие уже актуальны локально.
- Ранее для обновления приходилось каждый раз проходить через общий preview/import без явного режима “только изменения”.
- Не было даже базового мониторинга последних синхронизаций и фонового обновления уже импортированных статей.

## Ожидаемый результат

- Можно быстро обновить только изменённые статьи одной кнопкой из списка интеграции.
- В preview сразу видно, что новая статья, что изменилось в Notion, а что уже актуально.
- Последний результат синхронизации отображается в UI.
- Уже связанные с Notion статьи автоматически подтягивают обновления по расписанию сервера.
