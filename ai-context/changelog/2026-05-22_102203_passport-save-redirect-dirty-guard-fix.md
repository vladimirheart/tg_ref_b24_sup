# 2026-05-22 10:22:03 - passport save redirect dirty guard fix

## Промт пользователя

`при попытке сохранить паспорт объекта путём нажатия на  "готово" а потом на "Сохранить", страница сообщает что я пытаюсь с неё уйти, и как следствие сохранить изменения не удаётся`

## Что изменено

- В `spring-panel/src/main/resources/templates/passports/new.html` добавлен helper `markPassportSavedState()`, который единообразно сбрасывает локальный dirty-флаг страницы паспорта и global dirty-state из `EditGuard`.
- В `savePassport()` этот helper теперь вызывается:
  - перед `window.location.href = /object-passports/{id}` после успешного создания нового паспорта;
  - после успешного сохранения существующего паспорта вместо разрозненного частичного сброса только `EditGuard`.

## Зачем

- Раньше успешный `POST /api/object_passports` завершался redirect-ом, но dirty-guard ещё был активен, поэтому браузер считал этот переход попыткой уйти со страницы с несохранёнными изменениями.
- Из-за этого пользователь получал ложное предупреждение именно в момент успешного сохранения нового паспорта.

## Результат

- После сохранения нового паспорта UI сначала переводит страницу в saved-state, а уже потом делает redirect на детальную страницу.
- Защита от потери несохранённых изменений сохраняется для реального ухода со страницы, но больше не мешает штатному post-save переходу.
