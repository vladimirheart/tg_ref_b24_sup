# Notion Callouts And Stable Inline Media

## Что сделано

- Добавлена предобработка `callout`-блоков из Notion перед markdown-рендером.
- `callout` теперь отображаются как отдельные стилизованные блоки с иконкой и цветовой темой, близкой к Notion.
- Подмена inline-media на локальные `/api/attachments/...` переведена на стабильный ключ по URL без временных query-параметров.
- Для уже импортированных статей добавлен fallback по `originalName`, чтобы картинки находились даже если старое имя файла было рассчитано по signed URL.
- Обновлены CSS-стили knowledge markdown для новых callout-блоков.
- Добавлены тесты на рендер `callout` и на rewrite картинок по fallback-матчингу.

## Почему

- В экспортированных из Notion статьях встречаются блоки вида `<callout icon="❗" color="gray_bg">...</callout>`, которые раньше попадали в интерфейс как сырой текст.
- Встроенные картинки из `prod-files-secure.s3...` используют временные signed URL, и прямой матчинг по полному URL ломал отображение после истечения query-параметров либо при различии URL из markdown и block API.

## Ожидаемый результат

- `callout` в карточке статьи отображаются как оформленные визуальные блоки с иконкой.
- Inline-картинки и другие медиа чаще подменяются на локальные вложения, а не тянут уже протухшие ссылки Notion.
- Для старых статей часть картинок начнёт отображаться сразу после перезапуска панели, а для полной синхронизации медиа рекомендуется повторный импорт статьи.
