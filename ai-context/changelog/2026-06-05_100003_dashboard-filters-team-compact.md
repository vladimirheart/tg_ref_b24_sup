## Что изменено

- в dashboard перестроен layout блока фильтров: ресторан перенесён под период, а кнопки действий вынесены в отдельную компактную колонку;
- для `restaurantFilter` добавлено обновление доступных ресторанов по выбранному периоду через новый endpoint `/api/dashboard/restaurant-options`;
- основной payload dashboard теперь тоже возвращает `available_restaurants`, чтобы после применения фильтров фронт синхронизировал список ресторанов с текущим периодом;
- multi-select ресторанов уплотнён через стили `select2`, чтобы выбранные значения не растягивали фильтр по высоте;
- логика пресетов периода вынесена в явные helper-функции, и `Сбросить` теперь корректно возвращает активную плашку текущего месяца;
- большой блок `Команда` на dashboard уплотнён через более компактные тексты, отступы и меньшую высоту списка.
