# Changelog 2026-04-13 19:27

## Что изменено

1. Уточнён источник клиентского идентификатора.
- В карточке клиента и списке клиентов убрано жёсткое допущение про `ID Telegram`.
- Теперь UI показывает реальный источник клиента и корректную подпись идентификатора для `Telegram`, `VK`, `MAX` или внешней формы.

2. В диалоге появилась подгрузка предыдущих обращений клиента.
- Добавлен API для получения предыдущей переписки по тому же `user_id`.
- В модалке диалога сверху появилась кнопка `Загрузить предыдущие сообщения`.
- Архив предыдущих обращений подмешивается перед текущей историей и выделяется отдельным цветом.

3. В переписке и рабочих областях начали отображаться аватары.
- Для сообщений оператора используется аватар и display name текущего сотрудника.
- В модалке диалога и workspace-представлении добавлены аватары клиента, оператора и системных сообщений.

4. Исправлена ошибка совместимости со старой схемой `users`.
- По логам найдено регулярное падение на `no such column: is_blocked`.
- `AlertQueueService` и `NotificationService` теперь сначала читают схему через `PRAGMA table_info(users)` и строят безопасный SQL даже для старых БД.

5. Дополнительно восстановлена часть тестовой совместимости.
- В `DialogListItem` добавлен backward-compatible конструктор со старой сигнатурой без `resolvedAt`.
- В `SettingsBridgeController` добавлена перегрузка `updateItEquipment(...)` без `Authentication` для старых тестовых вызовов.
