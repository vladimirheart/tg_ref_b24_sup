# Changelog 2026-04-13 16:54

## Что изменено

1. Убран лишний клиентский ACK в MAX.
- Для активной заявки `bot-max` больше не шлёт клиенту служебное сообщение
  `"Сообщение добавлено в заявку ..."`.
- Сообщение клиента по-прежнему попадает в историю, активность заявки обновляется,
  операторский чат получает уведомление.

2. Автозакрытие переведено на реальные настройки проекта.
- `MaintenanceTasks` теперь читает `auto_close_hours` из shared settings.
- `0` означает отключение автозакрытия.
- Если настройка отсутствует или невалидна, остаётся fallback `24 часа`.

3. Исходящие ответы оператора теперь продлевают жизнь активной заявки.
- После успешного текстового или медиа-ответа из панели обновляется `ticket_active.last_seen`.
- Это выравнивает поведение MAX-диалогов с реальной операторской активностью.

4. Для MAX включена отправка медиа из панели.
- В `DialogReplyService` добавлена цепочка MAX media reply:
  - `POST /uploads`,
  - бинарная загрузка файла,
  - `POST /messages` с `attachments`.
- Благодаря этому paste изображения из буфера обмена в dialogs composer теперь
  проходит и для MAX-каналов.
