# 2026-07-07 12:10:30 - DB path preference fix

## Связанные задачи

- `01-143` - Переименовать канонические SQLite-файлы под их архитектурный смысл и сохранить runtime-совместимость

## Пользовательский промпт

> при выполнении задачи 01-143 видимо не все пути были перезаписаны, т.к. очень многого в проекте не вижу - ни диалогов ни аналитики ни клиентов - нет ни одной записи. самих ботов и их мониторинг тоже не наблюдаю

## Затронутые файлы

- `ai-context/changelog/2026-07-07_121030_db-path-preference-fix.md`
- `spring-panel/src/main/java/com/example/panel/config/EnvDefaultsInitializer.java`
- `spring-panel/src/test/java/com/example/panel/config/EnvDefaultsInitializerTest.java`

## Что сделано

- Исправлен выбор канонических SQLite-путей в `EnvDefaultsInitializer`, чтобы панель не цеплялась за первую найденную пустую копию БД из корня репозитория.
- Добавлено явное различение `workspaceRoot` и `panelHome`, чтобы при запуске из корня репозитория приоритетно рассматривался каталог `spring-panel`.
- Введён отбор лучшего существующего кандидата среди дублей по простому правилу: непустой файл предпочтительнее пустого, а при равенстве выигрывает файл большего размера.
- Добавлен тестовый сценарий на регрессию, где `spring-panel/panel_runtime.db` должен выигрывать у маленькой root-копии, а `panel_identity.db` из корня - у пустого module-файла.
- Для теста helper-методы выбора кандидатов открыты на package-level; неиспользуемый `resolveSiblingPath` удалён.

## Проверки

- Успешно выполнено: `spring-panel\mvnw.cmd -q --% -Dmaven.test.skip=true compile`
- Успешно выполнено: отдельная компиляция `EnvDefaultsInitializerTest.java` через `javac` с test classpath
- Успешно выполнена точечная проверка helper-логики через `jshell` на искусственном workspace-сценарии:
  - `runtimeOk=true`
  - `identityOk=true`
- Полный `spring-panel\mvnw.cmd -q "-Dtest=EnvDefaultsInitializerTest" test` сейчас не проходит из-за уже существующих ошибок компиляции в посторонних `Dialog*Test`, не связанных с этим фиксом

## Примечания

- Симптом пользователя объясняется тем, что после `01-143` в репозитории одновременно остались root-копии и `spring-panel`-копии SQLite-файлов, а прежний резолвер выбирал путь слишком рано и слишком грубо.
