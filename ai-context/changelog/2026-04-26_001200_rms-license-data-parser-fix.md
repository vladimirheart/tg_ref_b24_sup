# 2026-04-26 00:12:00 - RMS licenseData parser fix

## Что изменено

- Логика RMS-мониторинга переведена с поиска `<license>`-узлов на fallback-парсинг
  `licenseData`, где лицензии приходят как экранированный XML.
- Добавлен разбор `restrictionsByModule`, чтобы модуль `100` определялся по
  ключу `<k>100</k>` и брал срок действия из вложенного периода `<to>...`.
- Диагностический фрагмент лицензии теперь строится из распакованного
  `licenseData`, а не из сырого HTML-escaped куска ответа.

## Затронутые файлы

- `spring-panel/src/main/java/com/example/panel/service/RmsLicenseMonitoringService.java`
- `ai-context/tasks/task-list.md`

## Кратко

Причина ложной ошибки была в структуре ответа RMS: нужная лицензия видна не как
отдельный `<license>`, а как модуль внутри `restrictionsByModule`. После
исправления мониторинг может находить `id=100` и извлекать реальную дату
окончания именно для этой лицензии.
