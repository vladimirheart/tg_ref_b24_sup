## Что сделано

- В `IikoDepartmentLocationCatalogService` live-каталог перестроен на merge с уже сохранённой структурой:
  - активные департаменты из iiko продолжают приходить как live-источник;
  - ранее известные локации больше не исчезают из дерева, если они пропали из активного ответа iiko;
  - такие локации сохраняются в `tree`, но получают status `location::... = Закрыт`;
  - вернувшиеся обратно активные локации получают status `Активен`.
- Зафиксировано, что каскадные списки вопросов продолжают опираться на `statuses` и поэтому не показывают закрытые локации клиентам, хотя они сохраняются в `Структуре локаций`.
- В `IikoDepartmentLocationsSyncService` добавлен детализированный итог синхронизации:
  - общее число локаций;
  - число активных и закрытых;
  - сколько добавлено;
  - сколько закрыто текущей sync;
  - сколько переоткрыто;
  - короткие примеры добавленных, закрытых и переоткрытых локаций.
- В `SyncStatusSnapshot` добавлено поле `result`, которое сериализуется в API статуса sync.
- В `settings/index.html` рядом со status/progress блоком добавлен отдельный блок `Результат синхронизации`, который показывается для успешной sync и рендерит summary + примеры изменений.
- Обновлены тесты:
  - `IikoDepartmentLocationCatalogServiceTest` теперь проверяет сохранение ушедших fallback-локаций как `Закрыт`;
  - `IikoDepartmentLocationsSyncServiceTest` теперь проверяет payload успешной sync и её summary.
- Задача `01-077` переведена в статус `🟣`.
