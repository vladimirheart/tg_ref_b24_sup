# MCP Guide

Repo-local контекст по MCP всегда хранится в `ai-context/mcp/`.
Этот baseline-файл описывает только границу между общими правилами и
project-specific MCP-слоем.

## Что хранить в `mcp/`

- список MCP-серверов и коннекторов, реально используемых в этом репозитории;
- repo-specific инструкции по доступным сущностям, сервисам, командам и
  сценариям работы;
- договоренности о том, когда этот репозиторий должен использовать конкретный
  MCP, а когда нужно идти в код, docs или shell;
- рабочие playbook-файлы по интеграциям текущего проекта.

## Что не хранить в `mcp/`

- общие межпроектные правила по MCP, которые должны жить в
  `baseline/ai-rules/` и `baseline/guides/`;
- machine-specific токены, пароли и иные секреты;
- generated dumps, логи, кэш и временные артефакты.

## Где хранить секреты

Если для MCP нужны machine-specific доступы, они должны жить в
`ai-context/parameters/local-machine/`.
`mcp/` может ссылаться на такие параметры, но не должен содержать их реальные
значения.

## Рекомендуемый минимум

- `ai-context/mcp/README.md` - обзор каталога и список используемых MCP;
- по одному файлу на интеграцию, если проекту нужна детализация, например
  `github.md`, `figma.md`, `postgres.md`.

## Как использовать в задачах

- если задача зависит от repo-specific MCP, перед анализом подними применимые
  файлы из `mcp/` вместе с `rules/` и baseline-guides;
- в task-detail имеет смысл ссылаться на конкретные `mcp/*.md`, если они
  влияют на решение.
