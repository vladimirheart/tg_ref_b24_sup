# Changelog Guide

Живые changelog-записи всегда хранятся в `ai-context/changelog/`.
Этот файл описывает только baseline-правила ведения changelog.

## Формат

- одно завершенное изменение = один отдельный changelog-файл;
- рекомендуемое имя файла: `YYYY-MM-DD_HHMMSS_short-title.md`;
- после создания changelog-файл не дописывается другими изменениями;
- каждый changelog-файл должен содержать:
  - время;
  - короткий заголовок;
  - список измененных файлов или областей;
  - краткое описание фактически выполненных изменений.

Отдельный файл на каждое изменение снижает вероятность git-конфликтов.

## Пример

```md
# Переход на one-change-one-changelog

- Время: `2026-03-24 15:47:59`
- Файлы: `ai-context/tasks/task-list.md`, `ai-context/baseline/ai-rules/005_append-only-changelog.md`
- Что сделано: changelog переведен на модель «одно изменение = один файл».
```

## Исторические записи

Старые changelog-файлы, созданные по дневной схеме, можно сохранять без
массовой миграции.
