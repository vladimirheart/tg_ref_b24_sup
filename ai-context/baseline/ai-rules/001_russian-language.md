# Правило 001: Русский язык для документации и данных

## Статус

Действует.

## Правило

Документация, рабочие данные и сопроводительные материалы в репозитории должны
по умолчанию вестись на русском языке.

## Что входит в правило

На русском языке должны создаваться:

- README и project-документация;
- правила для AI-агента;
- task-описания и changelog-записи;
- markdown-артефакты контекста;
- аналитические и пояснительные материалы.

## Допустимые исключения

Можно оставлять без перевода:

- имена файлов и директорий, если английское имя технически удобнее;
- команды, пути, названия библиотек, API и протоколов;
- идентификаторы в коде;
- цитаты и фрагменты внешних источников, если важна точность оригинала.

## Практический смысл

Это правило делает всю рабочую базу знаний единообразной и естественной для
основного пользователя системы.
