# Правило 005: Любые file changes фиксируются в append-only changelog

## Статус

Действует.

## Правило

Если в рамках задачи были изменены, добавлены или удалены файлы проекта,
агент обязан создать отдельный changelog-файл в
`ai-context/changelog`.

## Обязательный подход

- одно завершенное изменение = один отдельный changelog-файл;
- рекомендуемый формат имени файла:
  `YYYY-MM-DD_HHMMSS_short-title.md`;
- после создания changelog-файл не должен использоваться для других изменений;
- старые changelog-файлы не переписываются без прямого запроса пользователя;
- changelog-файл должен перечислять фактические изменения, а не абстрактные
  формулировки.

## Практический смысл

История работы AI должна жить внутри репозитория как прозрачный и понятный
операционный журнал.

Отдельный changelog-файл на каждое изменение нужен еще и для того, чтобы
снизить вероятность git-конфликтов: разные изменения не дописывают один и тот
же общий файл.

## Исторические записи

Старые changelog-файлы, созданные по дневной модели, могут оставаться в
репозитории как история. Их не нужно массово перерабатывать, если пользователь
отдельно этого не просил.
