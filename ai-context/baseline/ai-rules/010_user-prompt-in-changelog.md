# Правило 010: В changelog нужно сохранять пользовательский промт

## Статус

Действует.

## Правило

Каждый новый changelog-файл, который создается после file changes, должен
содержать пользовательский промт, на основании которого было выполнено
изменение.

## Обязательный подход

- использовать явное поле `Промт пользователя:` или `Промты пользователя:`;
- если изменение было инициировано одним сообщением, фиксировать его дословно
  или с безопасной редакцией чувствительных фрагментов;
- если на результат существенно повлияли дополнительные сообщения пользователя,
  фиксировать и их тоже кратким списком в порядке поступления;
- если в промте есть секреты, токены, персональные данные или иная
  чувствительная информация, замещать только чувствительные фрагменты, не
  теряя смысл постановки;
- не придумывать промт задним числом: запись должна отражать реальную
  пользовательскую формулировку.

## Практический смысл

Фиксация пользовательского промта сохраняет контекст, который привел к
изменению, и помогает позже разбирать удачные формулировки как best practices.

## Исторические записи

Старые changelog-файлы можно оставлять как есть. Массово дописывать в них
пользовательские промты не нужно, если пользователь отдельно этого не просил.
