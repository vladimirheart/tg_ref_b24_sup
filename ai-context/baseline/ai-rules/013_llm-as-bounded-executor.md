# Правило 013: LLM используется только как ограниченный исполнитель над trusted evidence

## Статус

Действует.

## Правило

LLM в production AI-контуре не должен быть автономным decision-maker. Он может использоваться только как bounded layer:

- `parser` для вспомогательной интерпретации запроса;
- `rewrite` для безопасного retrieval-friendly переформулирования;
- `composer` для формулировки ответа на основе уже подтвержденного evidence;
- `explainer` для объяснения решения оператору.

При этом LLM не имеет права:

- принимать policy-решения вместо deterministic decision engine;
- обходить ограничения по sensitive/high-risk intent;
- отправлять ответ без trusted evidence;
- придумывать источники, ссылки, refund-обещания или PII-инструкции.

## Обязательные ограничения

- output guard обязателен для любого пользовательского текста, сгенерированного LLM;
- rollout LLM должен управляться feature flags и поддерживать безопасный откат;
- при любой ошибке LLM должен существовать deterministic fallback;
- policy boundaries и evidence requirements проверяются вне LLM.

## Практический смысл

Такой режим позволяет получать пользу от LLM в интерфейсе и формулировках, но не переносит на него ownership за бизнес-правила, безопасность и trust-модель. Это снижает риск hallucination-driven регрессий при развитии AI-контура.
