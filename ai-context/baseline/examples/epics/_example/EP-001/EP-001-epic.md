# Epic [EP-001] Повысить надежность checkout и обработку ошибок оплаты

## Цель

Снизить количество брошенных оплат, повысить наблюдаемость ошибок и упростить
диагностику проблем в checkout-потоке.

## Проблема

Сейчас ошибки оплаты обрабатываются неравномерно: часть случаев не
нормализована, пользовательские сообщения непоследовательны, а в логах не
хватает контекста для разбора инцидентов.

## Что должно получиться

- единая карта ошибок оплаты;
- предсказуемое поведение UI в типовых сценариях отказа;
- логирование и метрики, позволяющие разбирать инциденты без ручного
  воспроизведения;
- понятный handoff для backend, frontend и QA.

## Границы эпика

- backend-контракт ошибок checkout;
- frontend-обработка пользовательских сценариев;
- наблюдаемость и QA-покрытие критичных кейсов.

## Не входит

- смена платежного провайдера;
- редизайн всего checkout-flow;
- миграция на новую order-domain модель.

## Критерии готовности

- согласован список типов ошибок и их кодов;
- все критичные сценарии оплаты имеют ожидаемое поведение в UI;
- добавлены логи и метрики для ключевых отказов;
- есть тест-план и критерии приемки для QA.
