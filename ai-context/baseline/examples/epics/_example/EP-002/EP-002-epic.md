# Epic [EP-002] Пересобрать admin reporting v2 для продуктовой аналитики

## Цель

Собрать новый контур отчетности, который позволит продуктовой команде
самостоятельно анализировать воронку, retention и revenue без ручной
выгрузки данных разработчиками.

## Проблема

Текущие admin-отчеты нестабильны по структуре, не покрывают часть продуктовых
вопросов и требуют ручной интерпретации данных из нескольких источников.

## Что должно получиться

- набор отчетов с согласованной бизнес-терминологией;
- единые фильтры, периоды и правила агрегации;
- понятная схема handoff между аналитикой, backend и frontend;
- план внедрения без поломки текущих операционных отчетов.

## Границы эпика

- продуктовые отчеты уровня admin panel;
- подготовка backend-агрегаций и API;
- frontend-экран отчетов и критерии приемки.

## Не входит

- внедрение полноценного BI-контура;
- замена хранилища аналитики;
- миграция всех legacy-отчетов за один релиз.

## Критерии готовности

- согласован перечень отчетов первой поставки;
- у каждого отчета есть владелец, источник данных и definition of done;
- API и UI имеют общие правила фильтрации и агрегации;
- rollout-план учитывает обратную совместимость и риски.
